工作产品：设计类
该工件描述了一组共享相同的职责、关系、操作、属性和语义的对象。
用途

以下人员使用类：

  • 实施者，在实施类时将类用于规范。
  • 系统其他部件的设计人员用类来了解将如何使用他们设计的功能，及其这些功能的关系含义。
  • 用例设计者，以在用例实现中实例化它们。
  • 设计系统下一版本的人员，使用类来了解设计模型中的功能。
  • 测试类的人员，使用类来规划测试活动。

UML 说明 

名称  类名。  模型元素上的属性“名称”。 
简要描述  类的角色和目的的简述。  “短文本”类型的标注值。  
职责  由类定义的职责。  超类“Type”上的（预定义的）标注值。 
关系  类参与的关系，例如泛化关系、关联和聚集。  通过聚集“拥有”，由封闭包所拥有。 
操作  类定义的操作。  通过聚集“成员”，由超类“Type”所拥有。 
属性  由类定义的属性。  - " - 
特殊需求  文本描述，用于收集类上的所有需求（例如非功能需求），在设计模型中未考虑但需要在实施时考虑这些需求。  “短文本”类型的标注值。  
图  所有与类有关的图，例如交互图、类图或状态表图。  通过聚集“拥有”，由封闭包所拥有。 

可使用构造型以某种方式限定设计类或约束实施。例如，可以使用构造型表示该类代表特定编程语言构造。
