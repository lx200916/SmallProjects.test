活动：管理基线与发行版
此活动的目的是确保可针对各种用途将相关或从属工件的一贯集合确定为“基线”的一部分，如确定候选发行版、产品版本、工件完善性或完整性。

描述

设立基线的频率和正式程度在 CM 计划中有所描述。很明显，发布给客户的产品的正式程度要比内部项目团队中的可执行版本高得多。当工件合集达到某些完善阶段或级别时，则设立基线来协助管理可否发行、重用等等。

人员配备

这项工作主要由配置管理员角色带动，其中典型的需要是能够装配产品以供发行。发行的产品需要材料清单（BOM），以充当提供给客户的材料的完整清单。发行的产品将肯定需要部署活动中所述的发行说明和培训材料。

集成人员角色对这项工作的作用在于：确保开发人员工作空间提供的工件得以集成，这样它们就可并入可独立测试的工作版本。扮演集成人员角色者需要熟悉项目 CM 策略和测试实践。
使用
使用指导信息

这项工作通常始于精化阶段，而在构造和移交阶段最为普遍。在某些情况下，在先启阶段对工件设立基线可能是有价值的，但通常并非如此。可在单次迭代中和迭代期间的任意阶段创建和管理多个基线。

一般来说，如果不需要在核心开发团队以外对外交付可执行发行版，这项工作就是可选的。
