工作产品：集成构建计划
该工件为在迭代中的集成提供详细计划。
用途

集成构建计划的目的是定义组件的实施顺序、当集成系统时创建哪些构建，以及如何评估这些构建。

以下人员将使用集成构建计划：

  • 实施者，用它来规划实施设计元素的顺序、要交付到系统集成的内容以及交付时间
  • 集成人员，用它作为规划工具
  • 测试设计人员，用它来为迭代定义测试

关键注意事项

您应调整集成构建计划的大纲以适应您项目的性质。例如，如果是大型系统，则可能存在对每个实施子系统都有补充计划的情况。集成构建计划中包含或从该计划引用的此测试材料的正式程度和范围随该构建的重要程度而异。显然，应对照迭代规划中描述的所有评估条件来正式地评估迭代的最终构建。
