描述

此活动的目的是捕获项目经理每日持续进行的工作，包括：

  • 处理已由变更控制管理员核准的变更请求，并将这些变更安排到当前或将来的迭代中。
  • 持续地监视项目，从进度和质量的主动风险和目标方面进行评估；
  • 在状态评估中定期向项目复审委员会（PRA）报告项目状态，项目复审委员会是项目经理对其负责的组织实体；
  • 通过 任务：监视项目状态处理问题，并根据 工作产品：问题解决计划解决这些问题。对于项目经理无法单独授权的工作，可能需要发布变更请求。

人员配备

对于此部分的规程，项目经理需要具有组织、规划、沟通、时间管理、类选和分析等各方面的技能。管理复审员需要在项目管理方面具有很强的背景，需要深刻理解组织的业务业务策略和惯例，能够对项目的财务业绩和对应于合同义务的业绩作出判断。

使用
使用指导信息

此活动在每个阶段的每次迭代中进行。可能随时会出现问题。为符合项目的需要，其他活动的计时是定制的。例如，项目经理可以决定每周生成度量报告，每个月向 PRA 报告。

项目经理应该尽可能地使自动收集和减少项目信息（例如度量）的机制准备就绪。时间应该花在分析上面，而不是收集和计算上面。有关显然出现在项目中的问题的解决方案最终由项目经理负责。但是，有一类技术问题应该委托给软件设计人员，例如关于解决方案这方面的技术问题。然后项目经理来实施建议的解决方案，这可能会产生一个次要的问题，即资源缺乏，该问题是项目经理必须要解决的。这表明项目经理和技术人员之间必须存在某种信任 － 项目经理希望软件设计人员制定合理的技术解决方案，软件设计人员希望项目经理使基础结构和资源到位并实施它们，同时允许进行合同和财务约束。
