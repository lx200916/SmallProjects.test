描述

在此活动中：

  • 当实施者实施设计模型中的元素时，他们编写源代码、更改现有源代码、编译、链接并执行单元测试。如果发现设计中的缺陷，实施者将对该设计提交返工回馈。
  • 实施者还修订代码缺陷并执行单元测试以验证更改。最后，复审代码以评估质量以及与编程准则的一致性。

人员配备

由实施者执行的任务通常由单个人员完成。复审任务最好由一个跨功能小组成员组成的小型团队执行，通常其中有多名高级技术人员，他们对编程语言中所遇到的常见问题和缺陷有较多的经验。可能需要问题域中的特别专长，因为经常有系统涉及电话或带特殊接口的设备。 可能也需要有特定算法或编程技术的专长。

使用
使用指导信息

最好在几次会议中完成复审任务，每次会议侧重于系统的几个小的方面或侧重于特定问题。 这些会议的目标是确定代码中需要解决的特定问题，而不是在此时解决它们；应将解决方案讨论延迟到复审之后。频率较高、范围较小的复审比起频率较低、范围较大的会议，效率更高。
