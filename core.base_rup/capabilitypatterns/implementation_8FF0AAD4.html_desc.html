描述

为了帮助说明实施规程中涉及的工作，活动和工作产品将组织到规程的能力模式中。

每个活动代表一个高级别的目标，必须实现这样的目标才能有效地完成实施。在精化阶段早期完成构造实施模型。 对于在精化中开始的每次迭代，应规划集成、实施组件、集成每个子系统并最终集成系统。 后两个活动与集成测试活动紧密相关。

决定如何执行工作流程

应就实施规程的工作流程做出以下决策：

  • 查看此工作流程中的活动，然后决定如何执行工作流程。研究图及其警戒条件，以及下面的指南。决定要执行哪些活动并以何顺序执行。 
  • 决定要执行实施活动的哪些部分。以下是一些可以相互相对独立地引进的部分。

注释

集成和构建管理 角色集成人员和活动：计划集成连同工作产品：集成构建计划一起通常在项目早期引入。其他集成相关活动（例如任务：计划系统集成、任务：集成子系统和任务：集成系统）仅在集成开始之后适时地引入。 
实施组件 角色实施者和复审员以及他们的活动和工件是在每次迭代中实施开始时引入的。

  • 决定在项目生命周期内何时引入工作流程的每个部分。您常常可以等到精化阶段，然后再引入整个实施规程。 任何在先启阶段进行的原型构造通常是探索性的，它的进行（例如在工件和复审方面）不如精化和构造期间整个实施工作流程所要求的那样严格。
