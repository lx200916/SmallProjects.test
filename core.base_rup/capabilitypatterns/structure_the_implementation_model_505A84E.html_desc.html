描述

构造实施模型通常会产生一组实施子系统，可以相对独立地开发这些子系统。一个组织良好的模型将可以防止配置管理问题，并将允许从连续增大的集成工作版本中构建产品。

人员配备

虽然软件设计人员主要负责实施模型的结构，但是软件设计人员的经验还需要包括系统级别的集成者的经验。他们需要有软件构建管理、配置管理的经验，还需要有编写要集成的组件所使用编程语言的经验。因为集成者将处理自动化集成，软件设计人员不需要是脚本或集成自动化方面的专家，但熟悉一下该主题通常对更顺利地进行构建过程有所帮助。

使用
使用指导信息

应与体系结构其他方面的演进一起并行完成实施模型的构造；未能在体系结构设计过程早期考虑它将导致实施的组织很差，并可能阻碍实施和构建过程。在最坏的情况中，一个组织得不好的实施模型将阻碍项目团队并行开发软件。
