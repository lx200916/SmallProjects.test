活动：准备迭代环境
此活动的目的是为当前的迭代准备项目的开发环境，开发环境同时包含流程和工具。
扩展：准备迭代环境
描述工作分解结构团队分配工作产品使用
用途
此活动的目的是确保项目环境已准备好用于即将进行的迭代。 其中包含流程和工具。
关系
父代活动
  • 先启迭代
描述

对于每次迭代，此活动主要侧重于：

  • 完成开发案例，为进行迭代做好准备。
  • 准备和（如需要）定制要在迭代中使用的工具。
  • 验证工具是否已正确配置和安装。
  • 准备一组特定于项目的模板和指南，以支持在迭代中开发项目工件。
  • 确保适当地通知项目成员对项目环境所作的所有变更。

人员配备

流程工程师负责完成开发案例。单个人员通常不可能精通软件开发的所有规程，所以，流程工程师需要同时咨询开发组织内外部的专家，这一点非常重要。

特定于项目的模板和指南的开发可以由若干个人员共同完成。开发这些特定于项目的工件不需要对流程具有详细的了解。

通常会有若干个人员充当工具专家。每个人员负责一种工具或一组相关的工具。

使用
使用指导信息如其名称所示，此活动是在每次迭代开始时执行的。
关键注意事项

建议对于所有项目都进行此活动，虽然在某些类型的项目中所描述的任务可以使用不太正式方式执行，可以被考虑作为持续进行的项目环境支持的一部分。有关这种持续进行的支持的更多信息，请参阅活动：在迭代期间支持环境。
