活动：定义评估任务
此活动的目的是确定迭代的测试工作的适当重点，并与项目干系人在关于将指导测试工作的相应目标上达成一致。

描述

对于每次迭代，此项工作主要侧重于：

  • 确定测试工作的目标和可交付件
  • 确定一个好的资源利用策略
  • 为测试工作定义合适的范围和界限
  • 概述将使用的方法
  • 定义将如何监视和评估进度。

人员配备

虽然，测试规程中涉及的大多数角色参与执行此项工作，但该工作主要集中于测试管理员和测试分析人员角色。此项工作所需的最重要的技能包括协商、启发、策略和计划。

而此项工作的大多数资源将在构造阶段使用，从先启阶段到移交阶段需要为此项工作分配重要的资源。作为对于按阶段为此活动分配相关测试资源的相关指示，典型的百分比为：先启 － 50%、精化 － 25%、构造 － 10% 和移交 － 10%。
使用
使用指导信息

通常，在其他与测试相关的工作开始之前，在每次迭代开始时提出。

请注意，此项工作在每次迭代中执行。建议不要在执行测试任务的迭代之前过早地花大量时间来详细计划测试任务 － 一般来说，最多提早一次迭代来计划详细的测试工作。

执行此项工作的主要价值是考虑在迭代过程中将影响测试的各种因素和问题，并考虑应采取的适当操作。一般来说，不要将过多的时间用在关于测试工作的这些方面的文档陈述上。
