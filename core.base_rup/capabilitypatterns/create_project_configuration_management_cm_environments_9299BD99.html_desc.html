活动：创建项目配置管理（CM）环境
此活动的目的是建立一个环境，在其中可开发、构建整体产品，并使产品可供项目干系人使用。

描述

通过确保在各种专有和公共工作空间中开发人员和集成人员可按需要使用基本工件，以及随后充分设立基线和进行存储供以后使用，则可完成这项工作。设置 CM 环境包含创建 产品目录结构、存储库、 工作空间（开发人员和集成）和分配机器资源（服务器和磁盘空间）。

人员配备

要设置合适的环境，扮演配置管理员角色的人员需要很好地理解整体产品的构成，并将需要与软件设计人员密切合作，以确保设定足够的“占位符”CM 项。

在这项工作中扮演集成人员角色的人员需要确保开发人员工作空间提供的工件经过了充分测试，这样它们就可并入可测试的工作版本。集成人员角色需要熟悉项目 CM 策略、工作版本和测试实践。
使用
使用指导信息

该工作通常始于先启阶段，并延续到整个生命周期。一般情况下，这项工作在每个阶段之初（或每个前一阶段之末）的迭代中最为普遍，可为即将到来的阶段准备合适的 CM 环境。请注意，当团队规模扩大和缩小时，CM 环境的需求经常随阶段变化：较小规模的开发团队在早期阶段所要求的环境通常比在后期要求的环境正式程度要低。

此项工作不被视为可选的，虽然项目惯例预示这项工作将从很次要的注意事项转变为很重要的因素。请注意，如果现有的 CM 环境适当，这项工作可能更侧重于定制该环境，以适合该项目或生命周期阶段。
