能力模式：配置与变更管理
此能力模式涵盖配置与变更管理规程的活动和工作流程。
描述工作分解结构团队分配工作产品使用
用途

为了说明“配置”与“变更管理”规程中涉及的工作，活动、任务和工作产品被组织成规程的能力模式。

关系
环境
  • Classic RUP (for large projects)
描述

此能力模式中的每个活动代表一个高级别的目标，必须实现这样的目标才能执行有效的配置和变更管理。活动活动：计划项目配置与变更控制和活动：创建项目配置管理（CM）环境在项目启动时执行。其余的活动则在整个项目生命周期内持续地执行。

使用说明

决定如何执行工作流程

应就配置与变更管理规程的工作流程做出以下决策：

  • 查看此工作流程中的活动，然后决定如何执行工作流程。研究图及其警戒条件，以及下面的指南。决定要执行哪些活动并以何顺序执行。
  • 决定要执行配置与变更管理活动的哪些部分。 例如，只有在您打算系统地管理变更请求的情况下，才执行活动：管理变更请求。
  • 决定在项目生命周期内何时引入工作流程的每个部分。
