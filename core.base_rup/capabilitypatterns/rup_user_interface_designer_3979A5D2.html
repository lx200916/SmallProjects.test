角色：用户界面设计员
该角色协调用户界面的设计工作。这包括收集可用性需求和制作候选用户界面设计的原型以满足这些需求。

主要描述

“用户界面设计员”角色不负责实施用户界面。该角色致力于用户界面的设计和“可视化塑形”，所采用的方式如下：

  • 捕获关于用户界面的需求，包括可用性需求
  • 构建用户界面原型
  • 在可用性复审和使用测试会议中，考虑用户界面的其他项目干系人（例如用户）的需求
  • 复审并提供关于由其他开发人员（即设计人员和实施者）创建的用户界面的最终实施的适当反馈。

技能

用户界面设计员可以来自于创意和视觉艺术背景，而非商业、工程或计算机科学背景。用户界面设计员致力于系统的可用性。

分配方法

尤其在较大的项目中，通常都会建立单独的一组人员，该组中每个成员都扮演用户界面设计员角色。该组主要致力于用户界面和系统的可用性方面。这很重要，原因是：

  • 用户界面设计员所需的技能通常需要针对当前的项目和应用程序类型以及可能很独特的可用性需求进行改进和优化，而这既需要时间也需要精力
  • 必须限制“混合接合”的风险；即，用户界面设计员需要更多地考虑可用性而非实施
