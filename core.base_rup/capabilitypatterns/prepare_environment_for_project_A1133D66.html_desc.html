活动：准备项目环境
此活动的目的是准备项目的开发环境，开发环境同时包含流程和工具。
扩展：准备项目环境
描述工作分解结构团队分配工作产品使用
用途
此活动的目的是将底层的开发流程转变为特定于项目的流程，使开发基础结构为项目做好准备。
关系
父代活动
  • 准备项目环境
描述

准备开发环境涉及：

  • 定义项目将如何使用已配置的开发流程。
  • 按照时间和形式需求，限定需选择的工作产品。
  • 准备特定于项目的资产，如指南和模板。
  • 产生用于开发的候选工具列表。

人员配备

此活动中的任务最好由小型团队执行，配备具有良好的开发流程一般知识的员工。如果组织具有针对流程和开发环境的专门流程组，则角色：流程工程师从该组配备。

使用
使用指导信息

为项目准备环境已经在先启阶段完成。任何为改进和完成准备好的环境所做的后续工作都在“活动：为迭代准备环境”中描述。

关键注意事项

建议总是花一些时间来根据项目特定需求准备开发环境。

关于开发组织状态的背景信息通常在项目启动之前收集，记录在 工作产品：开发组织评估中。
