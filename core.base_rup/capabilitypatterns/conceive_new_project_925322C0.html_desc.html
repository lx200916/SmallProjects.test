活动：构思新项目
此活动的目的是将项目从初始的构想推进到一个流程点，在此流程点可以合理地确定是继续项目还是放弃项目。

描述

根据初始的远景，对风险进行评估和经济分析，然后生成业务案例。如果项目核准复审发现这些令人满意，则正式地设立项目（在任务：启动项目中），并作出有限度的核准（和预算）来开始完整的规划工作。请注意，按照定义，这个初始的远景是在项目范围之外创建的（可能是通过独立的业务建模或系统设计活动），而不是由后续的任务：制定远景在项目内创建。后一种活动用于向初始的远景中添加实质内容，验证并优化它。项目以此活动为起点，所以任何作为输入信息的工件必须已经存在，即项目必须具有特定的组织和业务环境。

人员配备

充当这些早期活动的项目经理的人员可能不会是展开项目的人员，因为此阶段的重点在于风险发现和确定可能的投资收益率。在这些早期活动中，项目经理作出良好的业务和技术风险判断的能力是非常宝贵的，而管理人员团队的能力则相对次要。同样，管理复审员也应具有大量的业务和领域经验。

使用
使用指导信息

应该执行此活动，因为知道是否向项目投资很重要。

在业务案例中，项目经理应该至少描述两种方法来实现远景，并根据风险影响和经济成果来分析它们。在项目核准复审期间，如果继续该项目，则将选择提供的选项中的一个。此时将有大量的管理知识和理论来帮助处于风险中的和进行决策分析的项目经理和项目复审员，让某些项目管理和复审人员精通这些技术是很有价值的 － 特别是在项目很大、无先例、很复杂或具有其他风险时。例如，请参阅 [CLE96]、[EVA98] 和 [VOS96]。
