活动：监视与报告配置状态
此活动的目的是通过持续的监视和报告来了解配置变更活动。
描述工作分解结构团队分配工作产品使用
关系
父代活动
  • 配置与变更管理
描述

此活动侧重于：

  • 确保工件及其相关基线可用。
  • 确定所需工件是否存储在受控库中和是否设立了基线。
  • 支持项目配置状态核算活动。
  • 促进报告变更请求信息，尤其是报告与针对缺陷和扩展请求执行的工作相关的活动。
  • 确保出于跟踪进度和趋势的目的对数据进行“汇总”和报告。

使用
使用指导信息

这项工作始于先启阶段，并适用于整个项目生命周期。

此项工作不被视为可选的，虽然项目惯例预示这项工作将从低正式度的任务变为高正式度的任务，并将增加复杂性，而项目团队规模会变大或分发更广。
