能力模式：[在范围内的] 测试和评估
此能力模式涵盖在特定范围内进行测试所需的活动。此范围可能是特定级别或类型的测试，例如功能测试。此范围也可能限制为在迭代过程中已实施或计划实施的组件或其中的部分。
描述工作分解结构团队分配工作产品使用
用途
使用该模式来在特定范围内进行特定级别或类型的测试，例如特定组件的功能、性能的测试或系统测试。
关系
环境
  • Classic RUP (for large projects)
描述
对要在迭代中执行的每种类型或级别的测试，该模式重复执行。

使用
使用说明此模式的名称中的 [在范围内] 表示此任务中的活动将局限于特定的范围，例如特定级别或类型的测试。此模式可使用多次，每次有相应的“范围”，例如，对于为迭代安排的每个级别或类型的测试（如性能测试、组件 X 的功能测试、系统测试），使用此模式一次。
