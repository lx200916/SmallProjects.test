活动：管理变更请求
此活动的目的是确保适当地考虑变更对项目的影响，并始终如一地在项目内进行经批准的变更。

描述

拥有标准、成文的变更控制流程，可确保在项目内以一致的方式进行变更，并向适当的项目干系人通知产品的状态、产品的变更和这些变更的成本和进度影响。

人员配备

变更（或配置）控制委员会（CCB）监督变更流程，该委员会由 RUP 中扮演各角色的代表组成。通常，这包括管理员、项目干系人（客户、最终用户）、开发人员和测试人员。在小型项目中，单个人（如项目经理或软件设计人员）可能是 CCB 方面的唯一代表。在 Rational Unified Process 中，这个主要 CCB 角色是变更控制管理员。
使用
使用指导信息

虽然这项工作始于先启阶段，并延续到整个生命周期，却会随着在生命周期中前进而愈加重要。在移交阶段进行管理通常比在先启阶段开始管理正式得多。

此项工作不被视为是可选的，虽然项目惯例预示这项工作将从很次要的非正式注意事项转变为更重要和更正式的因素。请注意，某些 CM 环境可能通过流程自动化（这种情况下将在某一工具内建立规则）来支持更改（或配置）控制板（CCB）功能。当必须在分散的各团队范围内管理 CCB 功能时，这尤其有价值。

可在 概念：变更请求管理中找到这些概念、建议任务和变更请求状态的进一步说明。
