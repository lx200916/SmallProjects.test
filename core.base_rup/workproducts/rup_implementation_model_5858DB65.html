工件：实施模型
实施模型代表实施的物理组成，这涉及到实施子系统和实施元素（目录和文件，包括源代码、数据和可执行文件）。
域：实施
工作产品类型：模型
用途

实施模型确定实施的物理部件，以便更好地理解和管理它们。实施模型定义了集成的主要单元（围绕这些单元组织团队），以及可以独立建立版本、部署和替换的单元。

更详细的实施模型可能还包括低级源代码和派生的文件以及它们与设计模型的关系。仅当在模型和文件之间有自动同步时，才建议这样的详细信息。

定制
说明选项

以 UML 表示：模型，构造型为 <<implementation model>>。 

实施模型可具有以下属性：

  • 简介：作为模型简介的文本描述。
  • 实施子系统：模型中的子系统，代表层次结构。
  • 实施元素：模型中的元素，由子系统所有。
  • 关系：模型中的关系，由实施子系统所有。 
  • 图：模型中的图，由实施子系统所有。
  • 实施视图：模型中的实施视图，它是一幅显示实施子系统和各个分层的体系结构视图。   

实施模型是可选的。如果选择创建实施模型，关键定制决策是如何关联实施模型和设计模型，以及哪些实施元素对模型足够重要。工作产品指南：实施模型中涵盖了如何作出这些决策的指导信息。 另请参阅技术：从设计映射到代码。
