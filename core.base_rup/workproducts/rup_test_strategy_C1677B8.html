工件：测试策略
该工件定义关于如何针对目标系统的一个或多个方面来进行测试工作的策略计划。
域：测试
工作产品类型：概念
用途
  • 将策略传达给外部项目干系人，以征得他们对测试方法的同意。
  • 将策略传达给测试团队的内部成员，以协调团队工作。
  • 使管理人员和其他项目干系人确信该方法是合理并且可以完成的。

描述
简述

测试策略记录以下信息元素：

  • 将要使用的常规方法的解释。例如，解释 主要方法如何基于针对需求或设计规范验证软件，如何基于针对故障模型试运行软件，如何基于使软件经受住已知攻击，或者如何基于某个其他一般方法。
  • 特定的类型、技术和 测试风格，它们将用作策略的一部分，并且对于每一项，是：
    • 技术的范围和适用性的指示。
    • 将如何使用技术的概述。
    • 要支持该技术将需要哪些工具的概述。
    • 度量成功的条件和使用技术的当前值
    • 技术的脆弱度或限制的指示，以及在哪些场合其他技术将代替该技术的指示。

请注意，对于给定环境（技术、领域等等）中的特定软件系统，策略可能在所有或部分后续开发生命周期中重用。

定制
说明选项

在某些测试文化中，测试策略被视为非正式的临时工作产品，而在其他测试文化中，它则是非常正式的，并且常常需要从外部停止。这样，其格式和内容必须是各式各样的，以满足项目或组织的特定需要。

作为正式文档的备选方案，您可以选择仅将测试策略的元素记录为一组非正式的计划说明，这些说明可能保存在测试团队可以轻松地看见并访问的内部网 Web 站点上或者白板上。
