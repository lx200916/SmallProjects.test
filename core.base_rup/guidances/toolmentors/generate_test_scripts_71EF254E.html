工具向导：使用 Rational TestFactory 实施生成的测试脚本
此工具向导描述如何使用 Rational TestFactory 自动生成测试脚本，这些测试脚本用于测试被测试应用程序的特定区域。
工具：Rational TestFactory

相关元素

主要描述

概述

“Pilot”是可用于生成测试脚本的 Rational TestFactory 工具。Pilot 生成用于测试被测试应用程序（AUT）中的控件功能的脚本，这些控件由应用程序图的某个区域中的 UI 对象表示。有关开发应用程序图的信息，请参阅工具向导：在 Rational TestFactory 中设置测试环境。

在运行 98/2000/NT 4.0 时此工具向导可用。

要使用 Rational TestFactory 自动生成测试脚本：

  1. 向应用程序图添加 Pilot
  2. 设置和运行 Pilot

1. 向应用程序图添加 Pilot

可在应用程序图中的任何位置添加 Pilot 对象。Pilot 将从此处自动生成测试脚本，这些测试脚本的重点放在与应用程序图的该部分相对应的 AUT 的特定功能区域。

要确定需要测试应用程序的哪一部分，请参阅工具向导：捕获自动化测试的测试设计结果中的第一个任务。

   请参考 Rational TestFactory 联机帮助中的以下主题：

  • Pilots: What they are and how they work
  • Effective Pilot placement
  • Add a Pilot

2. 设置和运行 Pilot

设置 Pilot 时，您需要指示控制 Pilot 如何运行的值和选项。可指定的设置信息包括：

  • Pilot 必须在应用程序图中测试的深度。
  • Rational TestFactory 用于停止 Pilot 运行的条件。
  • 要在测试中包含的应用程序图中的其他 UI 对象。
  • 要从测试中排除的 Pilot 控制下的 UI 对象。

在 Pilot 运行过程中，Pilot 将构建一个优化的“最佳脚本”，该脚本提供广泛的基于代码的测试覆盖率且不包含冗余脚本代码。

Pilot 还将生成针对基于 UI 的测试覆盖率优化过的“UI 脚本”。在 AUT 的后续构建中，可将 UI 脚本作为简单的冒烟测试运行来检查用户界面中的控件。

每次运行 Pilot 时，Rational TestFactory 都将在应用程序图的 Pilot 下添加一个新的“run”文件夹。该 run 文件夹包含 Pilot 生成的测试脚本。如果 Pilot 在运行过程中遇到缺陷，TestFactory 将把该缺陷测试脚本放到 run 文件夹下的“defects”子文件夹中。
