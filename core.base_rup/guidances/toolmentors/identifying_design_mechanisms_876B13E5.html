工具向导：使用 Rational XDE Developer 确定设计机制
此工具向导描述如何执行“使用 Rational XDE Developer 确定设计机制”。
工具：Rational XDE Developer
关系
相关元素
  • 确定设计机制
主要描述

概述 

此工具向导支持使用 Rational XDE Developer - Java Platform Edition 确定设计机制。

  • 此工具向导中的步骤与相应任务中的那些步骤相匹配。
  • XDE 联机帮助中的主题链接标有。
  • 在工具向导中，将执行以下步骤：
    • 对分析机制的客户端分类
    • 为用户体验机制确定设计机制
    • 列出实施机制清单
    • 将设计机制映射到实施机制
    • 记录体系结构机制

对分析机制的客户端分类

此步骤没有具体的 Rational XDE 指导信息。

为用户体验机制确定设计机制

此步骤没有具体的 Rational XDE 指导信息。

列出实施机制清单

对于列出所有可用的实施机制，没有提供具体的 Rational XDE 指导信息。 不过，以下 Rational XDE 联机帮助主题中描述了部分 J2EE 实施机制：

  • Understanding J2EE Pattern-Driven Development
  • J2EE Pattern-Driven Development
  • Supported J2EE Patterns

将设计机制映射到实施机制

此步骤没有具体的 Rational XDE 指导信息。

记录体系结构机制

机制本身是设计模型元素（如设计包、设计类和设计子系统），可在“工作产品：设计模型”中表示为各自设计活动的一部分。关于创建设计模型元素的指南，请参阅工具向导：确定设计元素。请注意，Rational XDE 模式特别适合记录设计和实施机制，因为它允许机制的客户端扩展模式并生成许多必需的设计和代码。
