工具向导：使用 Web Publisher 发布基于 Web 的 Rational Rose 模型
此工具向导描述了如何使用 Rational Rose Web Publisher 创建基于 Web（HTML）版本的 Rose 模型，其他人可以使用 Microsoft Internet Explorer 或 Netscape Navigator 等浏览器查看该模型。
工具：Rational Rose

主要描述

概述

通过发布基于 Web 版本的 Rose 模型，您能够使其他人以任意顺序浏览、搜索和导航您的模型。例如，您可以发布演进模型的连续迭代，以供复审或者共享信息。Rose Web Publisher 重新创建 Rose 模型元素，包括图、类、软件包、关系、属性和操作。一旦发布，超文本链接使您能够根据需要遍历 Rose 中的模型。您可以通过设置大量选项来控制 Rose Web Publisher 包含的内容。例如，您可以选择发布模型的哪些视图、要包含的详细程度、要使用的表示法以及 Rose 图的图形格式。请注意，如果您的模型用例附加有 RequisitePro 用例文档，那么 Rose Web Publisher 会保留这些文档，并在发布的 Web 页面上显示它们的 HTML 版本。

工具步骤

有两种发布模型的方法：

  1. 使用 Web Publisher 对话框
  2. 使用命令行批处理界面

1.  使用 Web Publisher 对话框

生成基于 Web 的模型的最简便方法是从 Rational Rose 的“工具”菜单使用 Web Publisher 外挂程序。Web Publisher 对话框提供了一个浏览器，您可以使用该浏览器选择要发布哪些模型元素。另外，您可以指定所发布的类是否应该包含超类信息以及图中使用哪种图形格式（如 .jpg 或 .gif）。“预览”功能使您能启动缺省浏览器，并从 Rose Web Publisher 直接查看所发布的模型。 

要发布模型，请执行以下步骤：

  1. 在 Rational Rose 中，打开要发布的模型。
  2. 从工具菜单中，单击 Web Publisher。
  3. 使用浏览器选择要发布的模型元素；选择要应用的发布选项。
  4. 当您准备好发布模型时，请单击发布。 
  5. 单击预览可打开您的缺省 Web 浏览器，并查看已发布的模型。 

有关 Web Publisher 的更多信息，请参阅联机帮助。

2.  使用命令行批处理界面

Rational Rose Web Publisher 具有命令行界面，通过该界面您可以使用批处理器发布模型。如果需要定期自动发布模型，这特别有用。您可以通过在初始化文件中定义发布参数来控制详细程度。当调用 Web Publisher 处理器时，请提供初始化文件的名称。处理器 rosewpbatch.exe 位于 Rose 安装目录下的 rosewp 子目录中。调用批处理器的语法是：

rosewpbatch.exe your_initialization_filename.ini 

示例：c:\rose\rosewp\rosewpbatch.exe c:\mybatchfiles\testbatch.ini

其中 your_initialization_filename.ini 是您创建的初始化文件的名称。在您对该处理器的调用中包含该 .ini 文件的完整路径。如果该处理器遇到错误，那么错误将记录在与该处理器位于同一目录的 rosewpbatch.err 文件中。您可以使用任何文本编辑器显示内容。

有关您在初始化文件中提供的参数和语法，请参考 Web Publisher 联机帮助。
