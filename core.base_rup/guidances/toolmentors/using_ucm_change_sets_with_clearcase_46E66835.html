工具向导：将 UCM 变更集与 Rational ClearCase 一起使用
此工具向导描述在统一变更管理（UCM）活动中使用变更集来创建和记录变更。
工具：Rational ClearCase
主要描述

概述

下图说明了 UCM 工作流程。此工具向导中讨论了加阴影的区域。

UCM 工作流程

术语注释：ClearCase UCM 活动紧密映射到 RUP 工作单。请勿将其与 RUP 概念活动相混淆。

UCM 活动跟踪处理开发任务时创建的版本。活动包括描述该任务的文本标题、活动创建者的用户标识和标识处理该活动时创建的所有版本的变更集。当准备就绪，可以修改源文件时，必须为活动设置开发视图。

在修改文件之前必须为开发视图设置活动。 

此工具向导在运行 Microsoft Windows 时适用。  

工具步骤

要处理 UCM 活动：

  1. 创建或查找并设置活动
  2. 检出和修改版本
  3. 检入工作

1. 创建或查找并设置活动 

当在 UCM 项目中检出文件或目录时，会请求您指定一个活动来跟踪工作。

创建新的活动并在当前视图中设置该活动

  1. 在“检出”、“检入”或“添加到源控制”对话框中，单击新建。
  2. 在“新建活动”对话框中，输入活动的标题。
  3. 单击确定。
  4. 要将此活动添加到活动流并在视图中设置此活动，请单击确定。

查找和设置活动

在工作会话之间保持活动。遵循这些步骤，查找现有活动并在开发视图中进行设置。 

从 ClearCase Explorer：

  1. 在 ClearCase Explorer 中，单击视图选项卡。
  2. 单击项目所在的页面并选择开发视图。 
  3. 在“文件夹”窗格中，单击我的活动。
  4. 在“详细信息”窗格中，选择要设置的活动旁边的复选框。

从 ClearCase 对话框：

  1. 从“检出”、“检入”或“添加到源控制”对话框执行以下操作之一：
  • 从“活动”列表中选择一个活动。
  • 单击浏览，并选择其他选择标准。
  1. 单击确定。ClearCase 将视图设置为在“活动”列表中当前所选的活动。

2. 检出和修改版本 

在修改源文件之前，转到“开发”视图并检出源文件。检出操作使文件或目录版本在视图中变为可写。  

  1. 在 ClearCase Explorer 中选择文件或目录，然后右键单击选中的文件或目录，并单击检出。 
  2. ClearCase 打开“检出”对话框，在“活动”列表中选中当前设置的活动。您可以接受该选择、选择另一个活动或创建新的活动。
  3. 在注释框中，描述计划好的变更。注释显示在版本的“属性浏览器”和元素的“历史记录浏览器”中。
  4. 执行以下操作之一：
  • 要只检出一个文件，请单击确定。
  • 要对所有选中项使用此对话框中的当前设置，请单击全部应用。
  • 要中断检出流程，并将正在检出的项和所有剩余项保留为检入状态，请单击取消。

3. 检入工作 

当想保留文件当前状态的记录时，请检入该文件。检入文件或目录会向 VOB 添加新的版本。版本信息由当前设置的活动来记录。  

在检入之后，视图保持设置为当前的活动。 

  1. 在 ClearCase Explorer 中，选择检出的文件或目录。 
  2. 右键单击选中的文件或目录并单击检入。
  3. 在注释框中，可以覆盖或补充在检出文件或文件夹时输入的注释。
  4. 执行以下操作之一：
  • 要只检入一个文件，请单击确定。
  • 要对所有选中项使用此对话框中的当前设置，请单击全部应用。
  • 要中断检入流程，并将正在检入的项和所有剩余项保留为检出状态，请单击取消。

请参阅 Developing Software 联机帮助，以获得各个步骤的详细信息。
