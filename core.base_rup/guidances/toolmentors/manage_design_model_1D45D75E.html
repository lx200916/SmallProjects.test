主要描述

概述

执行以下步骤可管理工作产品：设计模型：

  1. 创建设计模型
  2. 在设计模型中创建程序包
  3. 在设计模型中创建层
  4. 记录模型的组织

1. 创建设计模型

在 Rational Rose 中，工作产品：设计模型可以使用逻辑视图内名为“设计模型”的程序包表示。设计模型代表接近源代码的抽象级别。例如，设计模型中的类通常都分配了语言和模型属性，这些属性定义到代码的映射。

2. 在设计模型中创建程序包

程序包是将模型元素组织为组的通用模型元素。可将系统视为一个高级别程序包，其中包含系统内的其他全部内容。

有关程序包的更多信息，请参阅 Rational Rose 联机帮助中的 Package 主题。

3. 在设计模型中创建层

模型可包含若干层，可将这些层想象为程序包（每层一个程序包）。层可以嵌套。要区分出作为层的程序包，应向各程序包添加构造型“layer”。此外，还应使用“程序包规范”中的“文档”字段来描述程序包。

4. 记录模型的组织

每个模型都需要一个类图来显示其层和子系统的组织。该类图应为“主”图。请考虑将图重命名为“模型组织”。将代表层或子系统的程序包拖放到该图中。
