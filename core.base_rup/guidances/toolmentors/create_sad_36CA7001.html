工具向导：使用 Rational SoDA 创建软件体系结构文档
此工具向导描述如何使用 Rational SoDA 创建软件体系结构文档。
工具：Rational SoDA
主要描述

概述 

SoDA 使报告生成过程自动化，因而报告能快速并正确地创建。可使用 Microsoft® Word® 或 Adobe® FrameMaker® 版本的 SoDA 生成软件体系结构文档。 为了创建此报告，SoDA 将从 Rational Rose 模型收集体系结构上重要的各方面信息。仅当该模型遵守 Rose 模型的结构和命名约定时，这才有效。

相关的 Rational Unified Process 信息：

  •  报告：设计模型调查
  •  报告：用例实现
  • 工作产品：软件体系结构文档

在运行 Windows 2000、NT 4.0、Windows XP、Solaris 或 HP-UX 时此工具向导可用。

要使用 SoDA 创建软件体系结构文档，请使用与您的产品版本对应的过程：

  • 使用 Rational SoDA/Word 生成软件体系结构文档
  • 使用 Rational SoDA/FrameMaker 生成软件体系结构文档

使用 Rational SoDA/Word 生成软件体系结构文档

  1. 从 Rational Rose 的任何地方，单击报告 > SoDA 报告。
  2. 当 SoDA 中显示了可用报告列表时，请选择 Rational Unified Process 软件体系结构文档。
  3. 请单击确定以生成报告。

使用 Rational SoDA/FrameMaker 生成软件体系结构文档

  1. 从 FrameMaker 按钮栏，单击新建。双击 SoDA，然后双击 RoseDomain 并选择 RUPSoftwareArchitectureDocument.fm 模板。
  2. 编辑连接器并输入模型的名称。
  3. 请单击文件 > 另存为，将模板保存到个人目录或项目目录中。
    可能需要更改模板的名称以反映用例的名称；例如，ConductTransactionsReport.fm。
  4. 请单击 SoDA > 生成文档。
  5. 查看已生成的文档。

下一次您想生成此同一文档时，只要打开该文档然后单击 SoDA > 生成文档即可。

ROSE 模型的结构和命名约定

下面列出了 SoDA 从 Rose 模型抽取出来以包含到软件体系结构文档中的图：
  • 在体系结构上非常重要的用例和参与者在文档的“用例视图”部分显示。
  • 在体系结构上非常重要的类、接口、程序包和子系统在文档的“逻辑视图”部分显示。
  • 代表设计模型中的层的程序包在文档的“逻辑视图”部分显示。
  • “流程视图”程序包中的图在文档的“流程视图”部分显示。
  • “实施模型”程序包中的图在文档的“实施视图”部分显示。
  • “部署视图”中的图在文档的“部署视图”部分显示。
