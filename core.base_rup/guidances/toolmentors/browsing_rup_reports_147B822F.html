工具向导：向 ProjectConsole 导航树添加 Rational Unified Process 模板
本工具向导描述如何向 ProjectConsole 导航树添加 Rational Unified Process 模板。
工具：Rational ProjectConsole
主要描述

概述

ProjectConsole 利用 Java applet 技术为您提供了多种您熟悉的导航方式。如果直接单击树控制节点的文本，将显示与该节点关联的工作产品页面。如果通过单击节点的加号展开树控制节点，那么单击节点文本后会出现的超链接将显示为子节点，然后您可以从此处继续浏览。

此工具向导适用于 Microsoft Internet Explorer（发行版 5.5 或更高版本）或 Netscape（发行版 7.0 或更高版本）。

工具步骤

要向 ProjectConsole 导航树添加 Rational Unified Process 模板：

  1. 打开浏览器。
  2. 输入 ProjectConsole 站点的 URL。可联系 ProjectConsole 管理员以获得 URL。此时将显示 ProjectConsole 登录屏幕。
  3. 在此登录屏幕中输入有效用户标识和密码。此时将显示 ProjectConsole 工作产品浏览器。
  4. 选择父节点（即正在添加的 Rational Unified Process 模板之上的节点）。例如，要向 Analysis and Design 节点添加子节点，则展开以下节点：ProjectConsole、ClassicsCD.com Projects、Webshop、Release 1.0、RUP Disciplines 和 Analysis and Design。
  5. 右键单击父节点并从弹出菜单中选择添加。例如，右键单击 Analysis and Design 节点，然后选择添加。
  6. 在节点类型下的右框架中，单击报告。
  7. 在标签框中，输入报告的名称。请参阅下表的标签列。单击模板框旁的浏览按钮并浏览至 Rose 文件夹，然后选择相应的模板。请参阅下表中的模板列。单击确定。例如，在标签框中输入“Rational Unified Process Class Report”，并浏览至“Rose/rup_class_list_for_class_report”模板，以显示 Rational Unified Process Class Report。

    标签 模板
    Rational Unified Process 参与者报告 rup_actor_list
    Rational Unified Process 业务实体报告 rup_class_list_for_business_entities
    Rational Unified Process 业务对象模型调查 rup_business_object_model_survey
    Rational Unified Process 业务用例模型调查 rup_business_use_case_model_survey
    Rational Unified Process 业务用例实现报告 rup_use_case_list_for_business_uc_realization
    Rational Unified Process 业务工作者报告 rup_class_list_for_business_workers
    Rational Unified Process 类报告 rup_class_list_for_class_report
    Rational Unified Process 设计模型调查 rup_design_model_survey
    Rational Unified Process 程序包报告 rup_package_list_for_package_report
    Rational Unified Process 软件体系结构文档 rup_software_architecture_document
    Rational Unified Process 用例模型调查 rup_use_case_model_survey
    Rational Unified Process 用例实现报告 rup_use_case_list_for_uc_realization
    Rational Unified Process 用例报告 rup_use_case_list_for_uc_report
    Rational Unified Process 用户体验故事板报告 rup_use_case_list_for_user-experience_storyboard

  8. 单击图标框旁的浏览按钮，并浏览至“/projectconsole/pjc/Icons/Rose/Model.gif”，然后单击确定。
  9. 在工作产品标识框中，输入该 Rose 模型的路径。要使报告可显示有意义的数据，必须按照 Rational Unified Process 所述创建 Rose 模型。要使用样本 Rose 模型，请输入“Rose:Model:Path=${ARTIFACT_REPOSITORY}\Rose\ClassicsCDWorld.mdl”。

    注意：ClassicsCDWorld Rose 模型不符合 Rational Unified Process。

  10. 单击安全性列表旁的修改按钮，并添加应有权访问该报告的组和／或用户。单击确定。
  11. 单击保存。
  12. 现在，ProjectConsole 导航树中包含这个新的报告节点。单击该节点可显示报告。
  13. 要继续向导航树添加 Rational Unified Process 模板，请重复步骤 4 到 12。

注意：ProjectConsole 显示的报告是使用 Rational ProjectConsole Template Builder 创建的样本模板，发布的图表是在 ProjectConsole Dashboard 中创建的。
