工具向导：创建 Rational RequisitePro 项目的基线
此工具向导描述如何使用 Rational Administrator 和 Rational ClearCase 创建需求基线。

相关元素

主要描述

概述

在每个项目里程碑处创建需求项目的基线是一个不错的主意。基线创建的 ClearCase 对象代表 Rational RequisitePro® 项目在特定时间的稳定配置。该对象可用来创建新项目。

RequisitePro 项目由需求数据库、项目视图以及含有需求的文档组成。 可以直接在项目视图中创建需求，也可以在 RequisitePro 的 Microsoft® Word 文档中创建。所有需求均存储在数据库中，与它们在项目内所处的位置无关。您可以创建任何 RequisitePro 项目的基线，无论该项目使用的是 Oracle、Microsoft Access 还是 SQL Server 数据库。

注意：有关 RequisitePro 项目归档的信息，请参阅工具向导：使用 RequisitePro 归档需求。

使用统一变更管理创建项目基线

通过统一变更管理（UCM）模型，RequisitePro 项目管理员可以通过在 Rational Administrator 和 ClearCase 中执行任务来创建 RequisitePro 项目的基线。

在创建 RequisitePro 项目的基线之前，必须先满足以下前置条件：

  • RequisitePro 项目必须与 Rational Administrator 中支持 UCM 的 Rational 项目关联。
  • 必须在没有活动用户的情况下关闭 RequisitePro 项目。

创建 RequisitePro 项目的基线：

  1. 打开 Rational Administrator。
  2. 右键单击 Administrator 项目，然后单击配置。
  3. 在“配置项目”对话框的需求资产区域中，单击全部检入。
  4. 在随后出现的消息中确认您的选择，并输入用来描述基线的活动。
  5. 完成“全部检入”操作后，关闭“配置项目”对话框。
  6. 在 Rational Administrator 工具菜单中，单击 Rational ClearCase Project Explorer。此时会显示 ClearCase Explorer。
  7. 右键单击 RequisitePro 需求资产集成流，然后单击弹出菜单中的创建基线。
  8. 输入基线标题，选择完全基线类型，选择视图环境，然后单击确定。
