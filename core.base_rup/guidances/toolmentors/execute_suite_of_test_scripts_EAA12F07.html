工具向导：使用 Rational TestFactory 执行测试套件
此工具向导描述如何使用 Rational TestFactory 创建和运行测试套件。
工具：Rational TestFactory
关系
相关元素
  • 实施测试套件
  • 执行测试套件
主要描述

概述

“测试套件”是用于安排和执行测试脚本套件的 Rational TestFactory 元素。该 TestFactory 元素可用于组织测试脚本，将这些测试脚本作为套件以批处理的方式执行，然后分析其基于代码的测试覆盖率。

在运行 98/2000/NT 4.0 时此工具向导可用。

要使用 Rational TestFactory 执行测试脚本套件：

  1. 创建测试脚本的测试套件
  2. 执行测试套件

1.   创建测试脚本的测试套件

“测试套件”对象是将要作为批处理运行的测试脚本添加到的容器。测试套件可以包含 Rational Robot 中记录的测试脚本及 Rational TestFactory 中生成的测试脚本。

  • 要在 Robot 中创建测试脚本，请参阅“工具向导：使用 Rational Robot 实施测试脚本”。
  • 要在 TestFactory 中生成测试脚本，请参阅工具向导：使用 Rational TestFactory 实施生成的测试脚本。

测试套件用于组织和执行各组脚本：最佳脚本、缺陷脚本和针对被测试应用程序（AUT）的特定构建而创建的脚本。测试套件中包含的测试脚本将始终链接到“测试套件”对象。可将测试套件作为将来执行 AUT 构建时的回归测试套件执行。

Rational TestFactory 提供两种创建测试套件的方法：

  • 可让 TestFactory 使用所选测试脚本自动创建测试套件。
  • 可手动将测试套件对象添加到应用程序图，然后使用测试脚本填充测试套件。

如果要分析套件中的测试脚本基于代码的测试覆盖率，必须在运行测试套件之前检测 AUT。仅当检测了该应用程序时，Rational TestFactory 才计算测试脚本基于代码的测试覆盖率。

  • Test Suites: What they are and how they work
  • Working with Test Suites
  • Viewing code coverage for a Robot script
  • Instrumenting the application-under-test

2.   执行测试套件

执行测试套件时，Rational TestFactory 按照各测试脚本在测试套件列表中的显示顺序执行这些测试脚本。如果测试脚本的顺序非常重要，则可在运行测试套件之前将这些测试脚本重新排序。

 请参阅 Rational TestFactory 联机帮助中的 Run a Test Suite manually 主题。
