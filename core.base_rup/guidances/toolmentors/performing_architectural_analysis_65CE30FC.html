概述

此工具向导支持用 Rational XDE Developer - Java Platform Edition 执行体系结构分析。  

此工具向导假定已根据 XDE 模型构造指南创建了一组结构化模型。

  • 此工具向导中的步骤与相应任务中的那些步骤相匹配。
  • 到 XDE 联机帮助主题的链接用 标记。
  • 在本工具向导中执行以下步骤：
    • 开发体系结构概述
    • 评估可用资产
    • 定义子系统的高级别组织
    • 识别关键抽象

    • 开发部署概述

开发体系结构概述 

Rational XDE 可用作绘图工具，用于生成描述体系结构概述的非正式图。

评估可用资产

架构设计师必须考虑复用已有的资产，包括现有的 Rational XDE 模型。

定义子系统的高级组织

在设计模型中添加软件包以反映您的分层策略。根据 Rational XDE 模型构造指南中给出的分层示例，步骤如下：

  1. 打开设计模型。
  2. 浏览到包含分层子软件包的软件包。
  3. 为每一层创建软件包，构造型为 <<layer>>。参考：Grouping Model Elements in Packages 、使用构造型工作 。

有关构造模型的指南，请参考以下白皮书：

  • Rational XDE 模型构造指南
  • 分层策略

识别关键抽象

将关键抽象记录在类图中，并带有每个类的简短描述。要执行该操作：

  1. 打开设计模型。
  2. 浏览到包含关键抽象的包。请参阅 Rational XDE 模型构造指南。
  3. 添加类图。请参阅 Adding Diagrams 。
  4. 将类添加到图中，构造型为 <<entity>>。参考： Adding Shapes to Diagrams、 Assigning Stereotypes 。
  5. 向每个类添加描述。请参阅 Documenting Model Elements .
  6. 可以选择将文档与类相关联。请参阅 Linking External Files to Model Elements 。
  7. 定义各类之间存在的所有关系。
    • 添加关联关系。请参阅 Adding Association Relationships 。
    • 指定关联关系的种类。请参阅 Specifying Kinds of Association Relationships 。
    • 添加泛化关系。请参阅 Adding Generalization Relationships 。

参考：Working with Class Diagrams 。

识别构造型交互

仅在“先启”过程中执行本任务时，才包括这一步骤。

本步骤的目的是确定系统中关键抽象之间的、这样的交互：这些交互可以表现系统中的重要种类的活动的特征，或者能够成为它们的代表。这些交互作为“用例实现”被记录。

有关在 XDE 中创建用例实现的指南，请参阅工具向导：使用 Rational XDE 执行用例分析。

开发部署概述

  1. 向部署模型添加部署图。请参阅 Adding Diagrams 。
  2. 向图中添加节点。请参阅 Adding Shapes to Diagrams 。
  3. 添加节点之间的关联。请参阅 Adding Association Relationships 。

参考： Working with Deployment Diagrams 。
