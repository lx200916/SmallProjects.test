工具向导：使用 Rational Robot 执行测试套件
此工具向导描述如何使用 Rational Robot 执行测试套件（回放一个或多个测试脚本）和分析测试执行的结果。
工具：Rational Robot
关系
相关元素
  • 执行开发人员测试
  • 执行测试套件
主要描述

概述

在运行 Microsoft Windows 98/2000/NT 4.0 时此工具向导可用。

要使用 Rational Robot 执行已记录或已编程的测试脚本，请回放该测试脚本。

  1. 回放脚本
  2. 查看 TestManager 日志中的结果
  3. 使用比较器分析验证点结果

1. 回放脚本

回放脚本时，Rational Robot 将重复您的已记录操作并自动执行测试。通过自动执行，您可以比手动测试更快、更彻底的方式来测试您应用程序的每个新构建版本。 这样可以减少测试时间，并且可以提高覆盖率和整体一致性。

测试脚本在测试生命周期过程中将回放若干次：

  • 测试实施 － 使用曾经记录的被测试应用程序的相同构建版本回放脚本，以验证这些脚本是否按预期工作。这将验证被测试应用程序的预期行为的基线。
  • 测试执行与回归 － 回放脚本以将被测试应用程序的最新构建版本与测试实施过程中建立的基线相比较。开发周期过程中对所有测试脚本不断进行的回放将展现上一个构建版本以来可能已引入该应用程序的所有差异。可评估这些差异以确定其是真正的缺陷，还是特意进行的更改。

回放之前，如果测试脚本在上次运行之后已经更改，则 Rational Robot 将自动对其进行编译。

有关为测试脚本回放准备测试环境的信息，请参阅工具向导：在 Rational Robot 中设置测试环境。

要回放测试脚本：

  1. 单击工具栏上的回放脚本按钮。将显示“回放”对话框。
  2. 输入测试脚本名称或从列表选择。

要更改“回放”选项，请单击 GUI 选项。

结束后，请单击确定。
有关回放选项的信息，请参阅工具向导：在 Rational Robot 中设置测试环境。

  1. 单击确定以继续。
  2. 如果显示了“指定日志信息”对话框，则执行以下操作：
  • 从列表选择一个构建版本。要新建构建版本，请单击右侧的构建版本按钮。
  • 从列表选择一个测试日志文件夹。要新建测试日志文件夹，请单击右侧的日志文件夹按钮。
  • 接受缺省的测试日志名称（该名称与测试脚本名称相同），或输入新名称。
  • 单击确定。
  1. 如果出现提示，询问您是否要覆盖日志，请单击是。

注意：要停止回放测试脚本，请按 F11 键。仅当回放面向对象的命令时，Rational Robot 才可识别 F11 键。低级别操作过程中，F11 键将不停止测试脚本回放。

2. 查看 TestManager 测试日志中的结果

回放完成之后，使用 TestManager 测试日志可查看回放的结果，包括验证点故障、过程故障、异常中断和其他任何回放信息。

要控制测试日志信息和测试日志的显示，可设置“GUI 回放选项”对话框的“日志”选项卡内的选项：

  • 要以回放结果更新项目，请选择输出回放结果到日志。
  • 要在回放后自动显示日志，请选择回放后查看日志。

如果未选择该选项，则可在回放之后通过单击工具 > Rational Test > Rational TestManager 显示测试日志。

如果回放测试脚本产生了错误，则必须识别出该错误的具体原因。

未能成功执行的测试脚本在测试日志的结果列中以单词 Fail 标识（显示为红色），或以 Warning 标识（显示为黄色）。要针对的执行故障包含若干类：致命错误、脚本错误和验证点故障。致命错误通常表明测试环境配置（而非测试脚本本身）存在问题。请调查测试环境配置，隔离故障并解决对应的故障问题。解决测试脚本错误可能需要进行一些测试脚本维护。验证点故障（在测试实施过程中）通常需要更改验证点参数，或指出记录脚本时与回放脚本时在目标测试项理想状态方面的差异。请参阅下一部分以了解如何分析验证点。

可以在日志事件属性对话框中查看与该错误条件有关的附加信息。通过选择测试日志事件并单击查看 > 属性可打开该对话框。

对于发生了故障（或警告）的每个测试脚本，请查看与该故障有关的信息，并且如果需要，请编辑测试脚本（请参阅上面的第 3 部分：编辑测试脚本（如果需要））。可以通过单击查看 > 脚本从测试日志打开测试脚本（和 Rational Robot）。

编辑测试脚本之后，应将环境重置为相应的初始状态（工具向导：在 Rational Robot 中设置测试环境）（请参阅步骤：回放脚本）。

3. 使用比较器分析验证点结果

使用比较器可分析验证点结果。要从 TestManager 测试日志打开比较器：

  1. 在“日志事件”列，选择验证点并单击查看 > 验证点。将显示相应的比较器。也可以通过双击“资产”窗格中的验证点，从 Rational Robot 打开比较器。但是，如果使用这种方法打开比较器，则只能查看基线文件。
  2. 将突出显示基线与实际结果之间的差异。
  3. 查看这些差异并确定相应的操作过程，包括：
    （注意：可用操作将根据验证点的类型而有所不同。）
  • 将属性复制到基线 － 复制当前的单个实际属性，并覆盖基线属性。
  • 将所有属性复制到基线 － 复制所有实际属性值，并覆盖所有基线属性。
  • 通过双击属性（菜单、值等）并输入不同的值，编辑该属性。
  • 编辑属性列表（以确定验证点应捕获哪些属性）。
  • 创建或修改掩码。
  1. 在比较器中完成了相应更改之后，请单击文件 > 保存基线以保存更改，然后单击退出按钮以关闭比较器。

编辑测试脚本或验证点之后，应将环境重置为相应的初始状态（工具向导：在 Rational Robot 中设置测试环境）（请参阅步骤：回放脚本）。
