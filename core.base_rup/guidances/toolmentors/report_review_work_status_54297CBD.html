主要描述

概述

ClearQuest 使用户易于指定在查询中要从 ClearQuest 数据库访存哪种记录。如果用户运行查询时未定义任何选择条件（过滤器），ClearQuest 会列出数据库中的所有记录。这里是典型查询的某些示例：

  • 实施者查询被分配给他们以待修正的缺陷，并查看列表顶部的高优先级缺陷。
  • 项目经理按记录的创建日期查询记录，以监视测试人员的进度。
  • 技术编写人员在编写发行说明时查询与项目关联的、未解决的记录。

ClearQuest 运行查询，然后显示结果集。团队成员检查结果集表格，并选择记录进行查看、修改或作为报告或图表的基础。

有两种 ClearQuest 查询：

  • ClearQuest 管理员为所有用户创建公用查询
  • 单个用户可以构建和保存私有查询

请参阅 ClearQuest 联机帮助 > 目录与索引 > Working with Queries。
