工具向导：使用 Rational XDE Developer 描述运行时体系结构
此工具向导描述如何执行“使用 Rational XDE Developer 描述运行时体系结构”。
工具：Rational XDE Developer
关系
相关元素
  • 描述运行时体系结构
主要描述

概述

此工具向导支持执行“使用 Rational XDE Developer - Java Platform Edition 描述运行时体系结构”。

  • 此工具向导中的步骤与相应任务中的那些步骤相匹配。
  • XDE 联机帮助中的主题链接标有。
  • 在本工具向导中执行以下步骤：
    • 分析并行需求
    • 确定流程和线程
    • 确定流程生命周期
    • 确定流程间的通信机制
    • 分配流程间的协调资源
    • 将流程映射到实施环境
    • 将设计元素映射到控制线程

分析并行需求

此步骤没有具体的 Rational XDE 指导信息。

确定流程和线程

如果存在应用程序并行，则必须确定控制线程（控制线程显示为活动类）。

  1. 确定活动类所属的程序包。浏览到该程序包。
  2. 向类图添加活动类。请参阅 Adding Shapes to Diagrams 。
  3. 将类标记为活动。请参阅 Making Classes or Objects Active 。
  4. 浏览至需要此活动类的用例实现。
  5. 使用时序图说明复杂的流程间通信和线程间通信。有关创建作为用例实现的一部分的时序图的指导信息，请参阅工具向导：使用 Rational XDE 设计用例。
  6. 指定操作的并行类型。 Specifying the Type of Concurrency for Operations 有关消息驱动的 EJB 的指导信息，请参阅工具向导：使用 Rational XDE 确定设计元素。

确定流程生命周期

添加代表流程和线程生命周期的时序图。每个流程或线程都必须出现在创建或破坏它的时序图中。有关创建时序图的指导信息，请参阅工具向导：使用 Rational XDE 设计用例。

将设计元素映射到控制线程

使用聚集关系可显示活动类的组合，并显示在活动类的控制下执行的非活动类。

  1. 浏览至包含该“流程视图”的“设计模型”程序包。请参阅 Rational XDE 模型构造指南。
  2. 向此程序包添加类图。请参阅 Adding Diagrams 。
  3. 将活动类以及这些活动类控制下的重要设计类拖放到类图上。请参阅 Adding Shapes to Diagrams 。
  4. 添加关联关系。请参阅 Adding Association Relationships 。
  5. 指定聚集。 Specifying Kinds of Association Relationships 参考： Working with Class Diagrams
