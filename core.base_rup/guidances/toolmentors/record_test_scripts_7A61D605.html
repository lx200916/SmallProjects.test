工具向导：使用 Rational Robot 实施测试脚本
此工具向导描述了如何使用 Rational Robot 来记录或编写测试脚本，以及随后如何通过编辑测试脚本扩展它们。
工具：Rational Robot
关系
相关元素
  • 实施开发人员测试
  • 实施测试套件
主要描述

概述

在运行 Microsoft Windows 98/2000/NT 4.0 时此工具向导可用。

要使用 Rational Robot 记录和扩展脚本：

  1. 在 Rational Robot 中记录测试脚本
  2. 插入验证点
  3. 在需要时编辑测试脚本

1. 在 Rational Robot 中记录测试脚本

在记录测试脚本时，Rational Robot 会记录：

  • 您与接受测试的应用程序交互时的操作。这些用户操作包括击键和鼠标单击，它们帮助您在应用程序中导航。
  • 您创建的验证点，用于捕获并比较有关特定系统组件状态的信息。验证点是测试脚本中的点，您在这一点确定是否应该确认构建过程中目标测试项的元素的状态或行为。在测试脚本记录期间，验证点捕获有关系统状态的信息，并将它存储为预期结果的基线。在测试脚本回放期间，验证点再次捕获相同的系统状态信息，并将它与基线中存储的信息进行比较。

记录的测试脚本确定目标测试项的预期行为的基线。在已修订的目标测试项的构件可用时，您可以对后续构件执行该测试脚本，通常，将在手动执行比较的时间段后在验证点比较系统状态信息。

在最初的测试脚本记录和随后的回放期间，Rational Robot 面向对象记录技术检查 Windows 层的系统状态信息。 因此，成功的测试脚本回放通常不需要依赖于绝对的屏幕坐标。 面向对象的记录使测试脚本与细微的用户界面更改分离，从而简化了测试脚本的维护。

要记录新的测试脚本：

1. 通过设置记录选项准备测试环境。请参阅工具向导：在 Rational Robot 中设置测试环境。
启动接受测试的应用程序。（在启动记录后，您可以通过在 Robot 中单击插入 > 启动应用程序来启动接受测试的应用程序。）
2. 单击 Robot 工具栏上的记录 GUI 脚本按钮。
3. 输入名称（最多 40 个字符），或者从列表中选择它。
4. 要更改记录选项，请单击选项。结束后，请单击确定。
5. 如果选择了先前定义或记录的脚本，那么您可以通过单击属性来更改属性。结束后，请单击确定。
6. 请单击确定以开始记录。 将发生以下事件：
  • 如果选择了已记录的脚本，Robot 会询问您是否要覆盖它。单击是。（如果记录到现有的 GUI 脚本上，则会覆盖该脚本文件，但所有现有属性的将应用于新的脚本。）
  • Robot 处于最小化状态（缺省行为）。
  • 悬浮的 GUI 记录工具栏出现。您可以使用该工具栏暂停或停止记录、重新显示 Robot 以及在脚本中插入功能。
7. 通过执行测试中标识的操作来执行测试，并插入所需的功能（如验证点、注释和计时器）。有关详细信息， 请参阅 Robot 联机帮助。
8. 必要时，从面向对象的记录切换到低级别的记录。
面向对象的记录检查 Windows GUI 对象和接受测试的应用程序中的其他对象，且无需依赖精确计时或屏幕坐标。低级别的记录通过屏幕坐标和精确计时跟踪详细的鼠标移动和键盘操作。
9. 完成后，单击 GUI 记录工具栏上的停止记录按钮。将发生以下事件：
  • 您记录的脚本出现在 Robot 主窗口的“脚本”窗口中。
  • 脚本中的验证点（如果有）出现在左侧的“资产”窗格中。
  • 脚本文本出现在右侧的“脚本”窗格中。
  • 在编译或回放脚本时，编译结果出现在“输出”窗口的“构建”选项卡中。
10. （可选）通过单击文件 > 属性设置脚本的属性。

2. 插入验证点

要在记录或编辑测试脚本时插入验证点：

  1. 执行以下操作之一：
  • 在记录时，单击“GUI 记录”工具栏上的显示 GUI 插入工具栏按钮。
  • 编辑时，则将指针置于脚本中， 并单击“标准”工具栏上的显示 GUI 插入工具栏按钮。
  1. 在“GUI 插入”工具栏上单击“验证点”按钮。

  2. 验证点用验证点类型命名（如果脚本中有多个相同类型，则需加上编号）。

  3. 相应地编辑名称。
  4. （可选）设置等待状态选项。
  5. （可选）设置预期的结果选项。
  6. 单击确定。
  7. 如果提示选择对象，则将对象查找工具拖到所需的对象上，释放鼠标左健，以选择该对象。（可选）可以通过单击浏览按钮从对象列表中选择该对象。在选择了所需的对象时，单击确定。

有关更加详细的信息， 请参阅 Robot 联机帮助。 另请参阅 Using Rational Robot 手册中的“Creating Verification Points in GUI Scripts”一章。

3. 如果需要，编辑测试脚本

通过手动编辑测试脚本的源代码或插入部分新记录，您可以编辑现有的测试脚本。例如，您可以：

  • 编辑测试脚本的文本（删除行、移动文本等）
  • 在现有测试脚本中插入一系列新的用户操作（选择菜单命令、单击按钮等）。
  • 在现有测试脚本中添加新功能（添加验证点、注释等）
  • 转至具有编译器错误的行（转至特定行号，查找下一处错误等）

有关详细信息， 请参阅 Rational Robot 联机帮助。 另请参阅 Using Rational Robot 手册中的“Editing, Compiling, and Debugging Scripts”一章。
