工具向导：使用 Rational Method Composer 创建方法配置
此工具向导描述如何使用 Rational Method Composer 创建方法配置。
工具：Rational Method Composer
关系
相关元素
  • 为项目定制开发流程
主要描述

方法配置是 Rational Method Composer（RMC）中包含的方法库的一个用户定义的子集。

使用 RMC 创建配置涉及作出下列决策：

  • 从 RUP 框架和任何包含的插件选择相关的方法程序包。
  • 取消选中／清除任何不需要的方法程序包。
  • 将视图定义到配置中可支持项目干系人在配置中采用不同的角度。 这些视图可用于浏览已发布的配置。

RMC 为特定项目环境提供了一组预定义的 RUP 配置。新建配置之前，请考虑将 RMC 中的一个预定义配置用作起点。选择并复制与项目特征最接近的预定义配置，并通过适当地选择和取消选择方法程序包进一步对其进行定制。出现在 RMC 中的每个方法元素都具有对它所包含的内容的描述，以及关于为何在配置中包含它的指导信息。 可在方法库的“配置”程序包中找到现有配置。有关浏览方法库的信息，请参阅工具向导：使用 Rational Method Composer 浏览方法内容。

除了可定制要包含到配置中的内容之外，还可以将视图定义到内容中，这些视图可用于对项目内的给定团队不显示不需要的方法元素。例如，基于角色的视图（如，开发人员不一定要看到与项目经理相同的详细信息）。

方法配置不仅可用于创建流程（请参阅使用 Rational Method Composer 开发流程），还可用于发布（例如，一个配置定义哪些元素以 HTML 发布，哪些不以 HTML 发布）。发布配置时，产生的 RUP Web 站点将仅包含选定的元素，而视图则显示为树形浏览器的单独实例（或选项卡）。有关更多信息，请参阅工具向导：使用 Rational Method Composer 发布方法配置。

有关使用 RMC 创建方法配置的更多信息，请参阅“创建方法配置”备忘单。可通过选择“帮助 > 备忘单”来访问所有的 RMC 备忘单。
