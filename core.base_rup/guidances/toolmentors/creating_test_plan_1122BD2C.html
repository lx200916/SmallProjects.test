工具向导：使用 Rational TestManager 创建测试计划
此工具向导描述如何使用 Rational TestManager 创建测试计划。
工具：Rational TestManager
关系
相关元素
  • 定义测试方法
主要描述

概述

此工具向导适用于 Windows 98/ME/XP/2000 和 NT 4.0 平台。

 测试计划包含 TestManager 所组织的一系列信息，所包含的信息是不断演化的。它表示已就测试的内容和时间达成一致并有所了解。一个项目可以有多个测试计划，它们代表一项测试工作的不同阶段或方面。请围绕功能集、功能和测试类型组织测试计划。

每个测试计划都可以包含多个测试用例文件夹和测试用例。测试计划定义了所有关键测试元素，
这些元素是以测试用例文件夹和测试用例所组成的层次结构来表示的。有关创建测试用例的更多信息，请参阅工具向导：使用 Rational TestManager 创建测试用例。

工具步骤

要创建测试计划，请执行以下步骤：

  1. 创建测试计划
  2. 设置测试计划属性

1. 创建测试计划

  1. 从“测试资产工作空间”的计划选项卡，右键单击测试计划，然后单击新建测试计划。
    仅测试计划的名称为必需字段。请根据测试的类型为测试计划命名；例如，“功能测试”。 
  2. 在名称字段中，输入测试计划的相应名称。单击确定。

  请参考 TestManager 联机帮助中的以下主题：

  • Test Planning（索引：test planning）
  • Test Plans（索引：test plans > creating）

2. 设置测试计划属性

除指定测试计划名称之外，随着测试计划的演进，还可以指定其他属性。例如，可指定测试计划所有者，添加到外部文档的指针，以及指定与测试计划关联的配置和迭代。测试计划将随着时间的推移而不断进行。测试计划是以迭代方式定义的不断演进的资产。测试计划有多个属性：
  • 测试计划的描述 － 此字段用于描述测试计划的范围；例如，“涵盖所有主要使用方案”。 
  • 测试计划的所有者 － 分配测试计划的所有权以构建团队，并确保测试计划这种资产应需要这些元素的人员的要求而成形。
  • 与测试计划关联的配置 － 指定需要执行测试计划以验证测试目标正确行为的硬件和软件配置。
  • 与测试计划关联的迭代 － 指定项目生命周期内的里程碑，这些里程碑定义希望何时执行测试计划的特定部分（测试用例集）。从而使您可以了解测试计划中的进度安排。
  • 与测试计划关联的外部文档 － 外部文档可能包含下列内容：
    • 资源约束 － 时间、预算、主体、技能集
    • 测试策略，用于概述要完成的测试类型（功能、性能、可靠性）
    • 测试策略的实施，包括自动化测试和手动测试，以及对当前工具集的测试
    • 项目的范围
    • 项目的工期
    • 预期的进度安排
    • 项目质量的定义
    • 可交付件
    • 项目度量的定义
