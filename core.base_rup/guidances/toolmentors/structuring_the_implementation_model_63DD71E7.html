概述

此工具向导中的步骤与相应任务中的那些步骤相匹配。对 Rational XDE™ 联机帮助中主题的引用以进行标记。

此工具向导假定您已按照 Rational XDE 模型构造指南 中的描述定义了实施模型的顶级结构。本工具向导中的步骤允许优化这一初始结构。

在本工具向导中执行以下步骤：

  • 建立实施模型结构
  • 调整实施子系统

  • 确定如何处理可执行对象（和其他派生对象）
  • 定义如何处理测试资产

建立实施模型结构

确定实施子系统后，可以将它们作为组件图中的包进行建模，以显示子系统之间的依赖关系。这些图定义了实施视图。要创建它，请执行以下步骤：

  1. 浏览到将包含实施子系统的模型和包。（在建议的 Rational XDE 模型构造指南 中，这是集成支持模型。）
  2. 创建提供实施子系统概述的组件图。请参阅 Adding Diagrams。
  3. 为每个实施子系统创建包。（请参阅 Adding Shapes to Diagrams。）   请注意，在 Rational XDE 中不对实施子系统进行特定支持，因此该图只是一幅图。
  4. （可选）如果在各种代码模型中，这些实施子系统与项目和／或元素之间的关系不明显，则可以创建对设计元素图的可跟踪性。请参阅 Adding Diagrams。

每个项目的代码模型必须按照 Rational XDE 模型构造指南进行构造。请参阅 对程序包中的模型元素进行分组。    也可以在文件系统创建目录，以代码创建软件包，同时在模型中创建相应的软件包。

调整实施子系统

此步骤没有具体的 Rational XDE 指导信息。

为每个实施子系统定义导入

在先前创建的“实施视图”组件图上捕获导入依赖关系。请参阅 Adding Dependency Relationships。  请注意，这些依赖关系只向开发人员提供指导信息。Rational XDE 并不执行强制或检查。

项目导入应该与这些依赖关系相匹配。

要从项目外部定义或修改 JAR 的导入，请在工具栏上单击窗口，单击首选项，然后单击 Java，组织导入。

从项目外部导入称为“引用模型”。请参阅 Reference Models。

确定如何处理可执行对象（和其他派生对象）

可执行程序和其他可部署文件可以建模为 UML 组件。请参阅 Rational XDE 模型构造指南中的部署支持模型。如果确定对这些文件建模，则应用以下步骤：

  1. 浏览到将包含这些文件的模型和包。（在建议的 Rational XDE 模型构造指南中，这是部署支持模型。）
  2. 创建组件图。请参阅 Adding Diagrams。
  3. 添加每个可执行程序和其他可部署文件（建模为 UML 组件）。请参阅 Adding Shapes to Diagrams。
  4. 组织到程序包中（如 Rational XDE 模型构造指南中所述）。请参阅 Grouping Model Elements in Packages。

请注意，对于可执行程序，没有双向设计支持，因此必须手动维护此模型。
