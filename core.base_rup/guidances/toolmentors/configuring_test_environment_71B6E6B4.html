工具向导：在 Rational Test RealTime 中配置测试环境
此工具向导描述如何在 Rational Test RealTime 中配置测试环境。
工具：Rational Test RealTime
关系
相关元素
  • 支持开发
主要描述

编译器、链接程序、调试器和执行环境的每种组合都需要一个定制的目标部署端口（TDP）。可以在 Test RealTime 项目内逐个节点地对该 TDP 进行其他修改。

Rational Test RealTime 目标部署端口技术确保了实际无限的构建环境和目标支持。这一灵活性非常关键，因为除非 Test RealTime 的测试与运行时分析功能使用被测试应用程序所用的相同编译器、链接程序、调试器和执行体系结构，否则将把该功能视为无效。

基本 TDP 适配通过 TDP 编辑器实现。这个单独的实用程序可用于指定对特定构建和执行环境的 Test RealTime 支持至关重要的信息。只有极为熟悉目标开发环境和执行平台的特征与复杂性的开发人员才能实现正确的 TDP 适配。适配可以是任何行为，如从简单列出标准编译器包含文件的路径，到定义堆管理函数以确保 Test RealTime 的内存概要分析功能可跟踪非标准的内存分配函数。

尽管这种基本适配对于使用 Test RealTime 的“组件测试”和“运行时分析功能”已经足够，仍然可以执行进一步的修改来适应特殊的执行需求，如修改用于运行时分析的检测的级别和类型。缺省情况下，每个 Test RealTime 项目都分配了一个从“项目创建向导”中所选的 TDP 派生的基本“配置”。子节点将从项目继承该配置。但仍然可以为各子节点创建定制配置；事实上，该定制配置可以基于完全不同的 TDP。

“目标部署端口”的基本适配与个别“配置”结合在一起，可以确保正确配置测试环境。

工具步骤

要配置测试环境以确保成功使用 Test RealTime：

  1. 使用 TDP 编辑器定制目标部署端口
  2. 使用 Test RealTime 的“配置设置”进行特定于节点的修改

使用 TDP 编辑器定制目标部署端口

TDP 编辑器可用于修改 Test RealTime 随附的目标部署端口的缺省设置，以及为尚不支持的开发和执行环境创建全新的 TDP。

TDP 存储在基于 XML 的文件中（带有 .xdp 扩展名）；这些文件位于文件夹 ….\Rational\TestRealTime\targets\xml 中。TDP 编辑器通过图形形式，按照一种有序而记载完备的方式组织这些 .xdp 内容，从而确保其易于操作和理解。TDP 编辑器的左侧列出了可能需要定制的各种类别。窗口的右上部包含帮助信息。 窗口的右下部用于输入实际的数据。

正确配置了 TDP 之后，按“保存”按钮将自动在文件夹 …\Rational\TestRealTime\targets 中创建一系列定制文件和文件夹。对 .xdp 文件所作的更改可以保存在 TDP 编辑器内，此时将正确覆盖目标文件。

要配置现有 TDP 或者新建一个：

  1. 在 Test RealTime 中，选择菜单项“工具 -> 目标部署端口编辑器 -> 启动”
  2. 选择菜单项“文件 ->打开”－ 将显示文件夹 …\Rational\TestRealTime\targets\xml 的内容
  3. 如果要配置现有目标部署端口，请选择相应的 .xdp file 并将其打开。
    如果要新建目标部署端口，请选择菜单项“文件 -> 新建”，然后选择相应的语言。
  4. 修改各种“基本设置”、“构建设置”、“库设置”和“解析器设置”以适应环境。在左侧窗口左键单击要修改的项，然后在右下窗口输入相应的值。通过右上窗口获取指导信息。
  5. 完成了配置 TDP 之后，请按“保存”按钮，或者选择菜单项“文件 -> 保存”。

有关详细信息，请参阅 Rational Test RealTime Target Deployment Guide。

使用 Test RealTime 的“配置设置”进行特定于节点的修改

在 Test RealTime 中创建项目时，必须选择目标部署端口。此 TDP 将成为项目“配置设置”的基础。 用户可使用此项目配置文件定义特定于节点的设置，这些设置执行下列操作：

  • 覆盖通过 TDP 编辑器输入的基本设置
  • 修改只能在 Test RealTime 本身内设置的运行时分析参数

通过使用“配置设置”，可以向组节点内的每个测试节点分配不同的 TDP，或向多个应用程序节点分配不同级别的检测。这种灵活性使您可以自由地同时测试和分析多种语言，或在一次运行过程中获取不同级别的信息。

要访问和修改 Test RealTime 项目的“配置设置”：

  1. 在 Test RealTime 中打开项目之后，选择菜单项“项目 -> 设置”。
  2. 移动生成的标题为“<Project Name> 配置设置”的窗口并调整其大小，以便自由地查看屏幕右侧的“项目浏览器”窗口。
  3. 左键单击“项目浏览器”中的任何节点可访问该特定节点的“配置设置”。
  4. 使用“配置设置”窗口可定义和／或覆盖现有设置。只要覆盖了设置，都将以粗体字列出设置名称及其类别名称。
  5. 按“应用”按钮保存修改。

“配置设置”修改中的其他有用方面：

  • 常规 -> 主机配置 -> 目标部署端口
    用于选择将对每个组、应用程序或测试节点使用哪个 TDP。
  • 常规 -> 目标部署端口 -> 名称
    用于为配置文件创建定制名称（该文件的缺省名称是在项目创建过程中选择的活动 TDP）。
  • 运行时分析
    各种设置，用于指定每个节点要使用的运行时分析的正确类型和级别。

有关详细信息，请参阅 Rational Test RealTime User Guide 的主题“Graphical User Interface-> Configurations and Settings”下的信息。
