ClearQuest MultiSite 能为您的公司做什么？

ClearQuest MultiSite® 为 ClearQuest 增加了强大的功能。有了 MultiSite，不同地点的开发人员可使用同一模式和用户数据库。每个地点（站点）都具有数据库的一个副本和各自的模式存储库。站点随时都可以通过发送更新数据包，将特定副本中所作的变更传播到其他站点。更新流程可以是自动的，或用命令手动启动。 

ClearQuest MultiSite 的控制方案防止了不同站点的冲突性记录变更。大多数 ClearQuest 对象均可由一个副本控制，并且只有该副本的用户才能修改记录。 MultiSite 的功能可用于记录的共享控制，这样，多个用户可以轮流处理同一个记录。 

您可以使用 ClearQuest MultiSite 提供的同步工具按站点连接允许的频率在副本之间发送变更。如果具有 IP 连接，则可以使用 MultiSite 的运送机制来频繁而自动地同步化副本，允许其他站点上的用户访问您的站点上的最新变更。 如果没有 IP 连接，则可以选择适于您的实施的其他同步方法（例如，电子邮件或 FTP）。

需要 ClearQuest MultiSite 吗？

ClearQuest 还允许您使用现有的查询生成报告。一个 ClearQuest 报告由一个查询插入报告格式（用第三方工具创建）组成。角色（如管理员）创建报告格式，然后在 ClearQuest 工作空间中保存该报告格式。要生成报告，选择一种报告格式，然后指定哪个查询将其结果插入报告格式。报告存储在工作空间中，并可随时运行。
