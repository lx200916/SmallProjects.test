主要描述

ClearCase MultiSite 能为您的公司做什么？

ClearCase MultiSite 产品是 ClearCase 的附属产品，用于向多个地点分配软件开发工作。例如，如果公司的开发人员分发在波士顿、旧金山和班加罗尔，您可以让每个开发人员在当地实时访问源代码存储库（已定版本的对象库或 VOB）。使用 MultiSite，每个站点都具有自己的 VOB 副本，并且所有开发人员都可以无冲突地并行处理源文件。

ClearCase MultiSite 的控制方案防止了不同站点的冲突性变更。大多数 ClearCase 对象均可由一个副本控制，并且只有使用该副本的那些开发人员才能修改对象。MultiSite 的功能可用于源代码元素的共享控制，这样，多个开发人员可以轮流处理同一个源文件。 

您可以使用 ClearCase MultiSite 提供的同步工具按站点连接所允许的频率在副本之间发送变更，而不是“对变更置之不理”。如果具有 IP 连接，则可以使用 MultiSite 的运送机制和可配置脚本频繁而自动地同步化副本，允许其他站点上的开发人员访问您的站点上的最新变更。如果没有 IP 连接，则可以选择适于您的配置的其他同步方法（例如，电子邮件、FTP 或磁带）。

需要 ClearCase MultiSite 吗？

如果需要在几个不同位置之间分配开发工作，并且开发人员远程访问 VOB 不切实际，则应考虑使用 ClearCase 和 ClearCase MultiSite。使用 MultiSite 的另一个好处是，您可以将其用作备份机制，方法是在站点的不同区域创建副本并经常使其与主副本保持同步。通过备份次副本而不是主副本，可以使开发工作不受主副本的阻碍而得以继续。另外，如果主副本的主机发生灾难性的磁盘崩溃情况，则可以起用备份副本，以避免开发工作的停止。
