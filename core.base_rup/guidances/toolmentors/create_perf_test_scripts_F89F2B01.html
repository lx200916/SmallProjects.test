工具向导：使用 Rational Robot 创建自动化性能测试脚本
此工具向导描述如何使用 Rational Robot 记录性能测试的自动化性能测试脚本。
工具：Rational Robot
主要描述

概述

对于性能测试，通常是通过将脚本记录到 Robot 来创建脚本的。当您记录虚拟用户脚本时，您与被测试的应用程序之间的交互将导致客户端和服务器之间产生特定于协议的通信。Robot 记录该通信，并在会话记录完成之后将该通信抽取为 VU 语言测试脚本。

然后，TestManager 中的性能测试套件将执行该脚本的多个实例，以便将装入应用到被测试的系统。

此工具向导适用于 Windows 98/ME/XP/2000 和 NT 4.0 平台。

工具步骤

要使用 Robot 记录虚拟用户脚本：   

  1. 开始记录虚拟用户脚本
  2. 插入计时器、块、注释和同步点
  3. 分割脚本或结束会话记录
  4. 编辑虚拟用户测试脚本（如果需要）

1. 开始记录虚拟用户脚本

当您记录虚拟用户脚本时，您与被测试的应用程序之间的交互将导致客户端和服务器之间产生特定于协议的通信。Robot 记录该通信，并在您结束了记录会话之后将该通信用于生成脚本。

  1. 通过设置虚拟用户记录选项准备测试环境。请参阅手册 Rational Robot User's Guide 中的“Setting Recording Options”。
  2. 单击 Robot 工具栏上的记录 VU 脚本按钮。 
  3. 输入会话名称（最多 40 个字符）。
  4. 请单击确定以开始记录。
  5. 然后，Robot 将最小化（缺省行为），并且将显示悬浮的会话记录工具栏。该工具栏可用于停止记录或分割脚本和重新显示 Robot。而会话插入工具栏则可用于将计时器、块、注释或同步点之类的功能插入脚本中。并且还可用于启动应用程序。 
  6. 在“启动应用程序”对话框中，指定要启动的可执行文件。此可执行文件应为客户机应用程序的可执行文件。对于 Web 应用程序，请使用浏览器访问 Web 站点。
  7. 单击确定以启动客户机应用程序。
  8. 按照测试用例所指定，开始与被测试的应用程序交互。将记录客户机与服务器之间的协议流量。

2. 插入计时器、块、注释和同步点

记录虚拟用户会话时，可以插入计时器、块、注释或同步点。请单击会话记录工具栏上的显示会话插入工具栏按钮，然后单击以下按钮之一：

  • 启动计时器：将显示“启动计时器”对话框，并提示您输入要启动的计时器的名称。请输入名称并单击确定或使用取消按钮来退出对话框。
  • 停止计时器：将显示“停止计时器”对话框，并提示您输入要停止的计时器的名称。请输入名称并单击确定或使用取消按钮来退出对话框。 

计时器是将一组客户机／服务器交互括起的“括号”。在 TestManager 中执行了性能测试套件之后，执行各计时器所需的时间将在“性能”和“响应”报告中显示。

  • 注释：将显示“注释”对话框，并提示您输入要添加到脚本的注释。请输入注释并单击确定或使用取消按钮来退出对话框。注释是脚本中的文本。 它们帮助脚本读者理解脚本，但不会影响脚本的执行。
  • 同步点：将显示“同步点”对话框，并提示您输入要添加到脚本的同步点的名称。请输入名称并单击确定或使用取消按钮来退出对话框。同步点是一种脚本功能，可用于在执行脚本期间协调多个虚拟测试员之间的操作。
  • 启动块：将显示“启动块”对话框，并提示您输入在脚本中要启动的块。请输入名称并单击确定或使用取消按钮来退出对话框。
  • 停止块：将显示“停止块”对话框，并显示当前块的名称。 
    单击确定或使用取消按钮可退出该对话框。

与计时器类似，块也是将一组客户机／服务器交互括起的“括号”。但与计时器不同的是，块中不包含初始用户思考时间。 此外，块中的所有命令标识均以块名为前缀（而在计时器中，所有命令标识均以脚本的名称为前缀）。

   请参阅 Robot 联机帮助中的以下主题： 

  • Creating a new Suite
  • Inserting Comments into a Script
  • Inserting a Block into a Script
  • Inserting a Timer During Virtual User Recording
  • Inserting a Synchronization Point During Recording

   请参阅 Rational Robot User's Guide 中标题为 Adding Features to Scripts 的主题。 

3. 分割脚本或结束会话记录

完成执行测试用例或用户活动的逻辑单元之后，可以分割脚本或结束会话记录。 

分割会话表明上次会话分割（或会话的开始）以来记录的所有内容都代表一个逻辑单元（如登录到数据库或修改特定记录）；例如，在客户关系管理应用程序中更新客户的信息。

  1. 要分割脚本：

  • 单击会话记录工具栏上的分割脚本按钮。 

  • 为刚记录的活动输入虚拟用户脚本名称（最多 40 个字符）。

  1. 要停止会话记录：
  • 单击会话记录工具栏上的停止会话按钮。 
  • 输入虚拟用户脚本名称（最多 40 个字符）。
  • 要更改记录选项，请单击选项。结束后，请单击确定。
  • 根据您以前对虚拟用户记录的设置，生成器将生成脚本（自动过滤），或提示您选择应当用于生成虚拟用户脚本的客户机服务器通信和协议（手动过滤）。

   请参阅 Rational Robot User's Guide 中标题为 Recording Sessions - Choosing the Protocols to Include in a Script 的章节。

4. 编辑虚拟用户测试脚本（如果需要）

可通过按照如下方式手动更改脚本的文本来编辑现有脚本：

  • 编辑脚本的文本（删除行、移动文本等）。
  • 向现有脚本添加新的用户操作（添加共享变量或函数）。
  • 向现有脚本添加新的功能（添加计时器、块、注释等）。

   请参阅 Robot 联机帮助中标题为 VU Language Reference 的主题。
