工具向导：使用 Rational Administrator 配置项目
此工具向导描述如何使用 Rational Administrator 配置管理员项目。
工具：Rational Administrator
关系
相关元素
  • 支持开发
主要描述

概述

可以在 Administrator 中创建存储软件测试和开发信息的管理员项目。您计算机上的所有 Rational 组件都将从相同的项目更新和检索数据。管理员项目中的数据类型取决于已安装的 Rational 软件。 

有关管理员项目的信息，请参阅 Administrator 联机帮助中的以下主题：

  • What is a Rational project?
  • Parts of a Rational project
  • Benefits of a Rational project

也可以使用 Administrator 为 UCM（统一变更管理）启用管理员项目，UCM 是管理对需求、设计模型、文档、组件、测试用例、源代码等的变更的 Rational Software 方法。 

有关 UCM 的信息，请参阅 Administrator 联机帮助中的以下主题：

  • What is UCM?
  • What UCM is in Relation to the Rational Administrator
  • Deciding whether to make the project UCM-enabled

必须首先创建项目，然后才能对其进行配置。创建管理员项目之前，需要解决一些关键问题。如果计划对项目启用 UCM，则还需设置 Rational ClearCase。 

有关创建管理员项目所需的步骤的信息，请参阅 Administrator 联机帮助中的以下主题： 

  • Before creating a project
  • ClearCase setup steps
  • Creating a Rational Administrator project

如果要使用现有存储库，则可将其转换为管理员项目，以便由 Administrator 的当前版本使用。 

有关转换的更多信息，请参阅 Administrator 联机帮助中的以下主题： 

  • Upgrading an SQA 6.x repository
  • Upgrading a Rational Test 7.x repository

在运行 98/2000/NT 4.0 时此工具向导可用。

要配置管理员项目：

  1. 决定项目中要包含哪些 Rational 数据结构
  2. 决定是否要将管理员项目与 UCM 项目关联（如果尚未执行此操作）
  3. 决定是否要将 Rational RequisitePro 项目或 Rational Test 数据存储放入配置管理下（如果尚未执行此操作）

1.   决定项目中要包含哪些 Rational 数据结构

可以将各种数据结构与管理员项目关联。这些数据结构为：

  • Rational Test 数据存储
  • Rational RequisitePro 项目
  • Rational ClearQuest 数据库
  • Rational Rose 模型文件

可以从 Administrator 内创建数据结构，或选择现有的数据结构，但 Rose 模型文件除外。只能从 Administrator 中添加 Rose 模型文件，而不能从中创建。

  请参阅 Administrator 联机帮助中标题为 Configuring a Rational Administrator project 的主题。

2.   决定是否要将管理员项目与 UCM 项目关联

创建了管理员项目之后，如果该项目是在未启用 UCM 的情况下创建的，则可对其启用 UCM，但仅当您的系统上安装了 Rational ClearCase 时才可执行该操作。也可以作为修改项目属性的一部分，对项目中的需求资产和测试资产启用 UCM。

请注意，不能禁用在 UCM 下具有数据存储的、已启用 UCM 的项目。这意味着除非测试和需求数据存储也都没有启用 UCM，否则不能将已启用 UCM 的项目的状态更改为未启用 UCM。

   请参阅 Administrator 联机帮助中标题为 Associating a Rational Administrator project with a UCM project 的主题。

3.   决定是否要将 Rational RequisitePro 项目或 Rational Test 数据存储放入配置管理下

如果已经将一个 RequisitePro 项目或 Test 数据存储与启用了 UCM 的项目关联，但尚未将其放入配置管理下，则可执行该操作。

同时，如果在“配置项目”对话框中选择了 ClearQuest 数据库，也将为变更请求管理启用与该管理员项目关联的任何 UCM 项目。

但是，如果 ClearQuest 数据库已经与 UCM 项目关联，则不能选择或创建另一个 ClearQuest 数据库来与该管理员项目关联。要创建或更改与管理员项目关联的 ClearQuest 数据库，必须新建一个项目。此 ClearQuest 数据库由与 UCM 项目之间的现有关系预先确定。

在已启用 UCM 的管理员项目中，可以解除配置管理下的 RequisitePro 项目或 Test 数据存储的关联。请注意，从配置管理除去 RequisitePro 项目或 Test 数据存储将执行以下操作：

  • 将 RequisitePro 项目和 Test 数据存储中的所有文件复制到非版本控制区域
  • 将项目与非版本控制区域关联

   请参阅 Administrator 联机帮助中标题为 Configuring a Rational Administrator project 的主题。
