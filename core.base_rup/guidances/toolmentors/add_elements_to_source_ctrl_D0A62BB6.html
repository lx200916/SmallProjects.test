工具向导：使用 Rational ClearCase 向源控制添加元素
本工具向导描述如何使用 Rational ClearCase 向源控制添加元素。
工具：Rational ClearCase
主要描述

概述

处于 ClearCase 核心的是其广泛的控制和管理元素的版本的能力，这些元素诸如软件的源文件以及文档、目录、报告等等。在您能检出一个或多个元素之前，必须首先使这些元素的初始版本能进行源控制。

工具步骤

要使用 Rational ClearCase 向源控制添加一个或多个元素，请执行以下步骤：

  • 启动 ClearCase Explorer
  • 浏览至要放入源控制的元素
  • 选择元素
  • 向源控制添加元素
  • 完成源控制操作

    更多信息

启动 ClearCase Explorer

  1. 从开始菜单，单击程序 > Rational Software > Rational ClearCase > ClearCase Explorer
  2. 选择指向 VOB 的视图，您想要将 VOB 中的元素添加至源控制。


  3. 如果视图不存在，必须首先创建视图。如果视图存在，但您看不到 VOB，则必须首先安装将包含元素的 VOB。

浏览至要放入源控制的元素

我们建议您选择离根目录树最远的元素。对于任何给定的元素，ClearCase 也添加所有尚未处于源控制下的父目录（最高至 VOB 根目录）。

注意：在向 VOB 添加元素之前，必须首先将其父目录检出到视图。如果您尚未执行此操作，ClearCase 会在创建元素之前将其检出，并在创建元素之后将其检入。

选择元素

  1. 可以使用标准选择方法，选择要置于源控制下的一个或多个元素。

向源控制添加元素

  1. 右键单击选定的文件并从弹出菜单选择添加至源控制。
  2. 要向统一变更管理（UCM）项目添加元素，可在“添加至源控制”对话框中选择或创建 ClearCase 活动。
  3. 在“注释”框中，描述要添加至源控制的元素。
  4. 要在将新元素添加到源控制之后继续处理新元素，请选择在添加到源控制之后检出选项。要保留它们为检入，则清除该选项。
  5. 如果您正在复制的 VOB（多站点）中工作，且不在使用 UCM 视图，那么您可以使当前副本控制添加至源控制操作期间创建的所有分支，方法是选择使当前副本成为所有新近创建的分支的主副本选项。创建元素之后，您将能在任何新分支上创建元素的新版本。

  6. 如果不选择此选项，那么元素的主要分支的控制将指定给控制主要分支类型的副本。如果主副本不是当前副本，那么您不能在分支上创建任何新版本。另外，如果您的配置规范包含“自动生成分支”规则，那么在当前副本不控制分支类型的情况下，分支的创建会失败。

完成源控制操作

如下完成操作：

  1. 要只向源控制添加一个元素，请单击确定。
  2. 要向源控制添加多个元素，并应用相同的注释以及检出到所有元素，请单击全部应用。
  3. 要取消操作，单击取消。元素将不会置于源控制下并保持视图私有文件。
