工具向导：使用 Rational ClearQuest 引发项目干系人请求
此工具向导描述如何使用 Rational ClearQuest 提供的各种功能捕获和管理项目干系人请求。
工具：Rational Suite AnalystStudio
主要描述

概述

Rational ClearQuest 通过提供完整且一致的改进请求表单，帮助分析人员有效地收集项目干系人请求。该表单可针对您的开发工作场所按需定制。所有请求都记录在一个集中的变更请求数据库中。内部项目干系人（组员）使用 ClearQuest 窗口或 Web 应用程序提交改进请求；外部项目干系人可以使用 ClearQuest Web 应用程序。改进请求提交之后，分析人员可以通过向每个请求添加信息来进一步限定这些请求。

工具步骤

此文档包含以下步骤：

  1. 使用 ClearQuest Web 记录改进请求
  2. 在 ClearQuest 窗口中组织改进请求
  3. 运行查询

有关配置和定制 ClearQuest 表单的详细信息，请参阅 ClearQuest 联机帮助和 Rational ClearQuest Administrator's Guide。

1. 使用 ClearQuest Web 记录改进请求

ClearQuest 提供了一个 Web 应用程序，可配置该应用程序来向非 ClearQuest 许可证持有者提供受限功能。具有受限访问权的 ClearQuest Web 用户可提交变更请求，并有权访问管理员定义的单个 ClearQuest 查询。这项独特的功能可使不属于正在构建系统的软件团队的关键项目干系人使用 Web 提供反馈。此外，这些项目干系人仍然可以了解请求的进展情况，并可通过运行有限访问查询来检索更新的状态。

要使用 ClearQuest Web 提交改进请求：

  1. 启动浏览器并输入 ClearQuest Web URL。将显示 ClearQuest 登录页面。
  2. 输入您的用户名和密码，然后单击登录按钮。即连接到 Web 上的 ClearQuest 数据库。
  3. 从屏幕左侧的“目录”列表，找到操作标题并单击子标题提交记录。将显示选择记录类型页面。
  4. 在记录类型列表中，选择改进请求并单击提交按钮。将显示提交改进请求表单；该表单由三个部分组成。
  5. 在主要部分的标题、原因、客户优先级和描述框中，输入信息。
  6. 在记录的附件部分中，通过单击添加按钮并链接至外部文件，附加改进请求的任何可用支持信息。
  7. 在第三部分联系信息中，输入所有适用的客户与提交者信息（姓名、公司、电子邮件地址）。
  8. 单击确定。将创建改进请求并将其添加到中央数据库中。

有关配置 ClearQuest Web 以向非 ClearQuest 许可证持有者提供受限功能的详细信息，请参阅 ClearQuest Web 联机帮助中的 Editing Web Settings。

2. 在 ClearQuest 窗口中组织改进请求

改进请求提交之后，ClearQuest 将提供按选项卡组织的更多字段，以补充记录表单上最初提交的信息。分析人员可使用分析选项卡、解决方案选项卡和注释选项卡，将提交者信息与特定于团队的信息（如营销优先级和目标发行版）进行平衡，以达到组织的目的。分析人员使用这些信息来查询改进请求数据库，以便确定各发行版中要实施哪些请求。

要使用 ClearQuest 窗口组织改进请求：

  1. 选择改进请求并单击“改进请求”表单右侧的操作 > 修改。此操作将激活该请求，使您可以对该记录进行编辑。
  2. 单击分析选项卡以将其激活。
  3. 通过单击各框（营销优先级、请求类型、产品、产品区域、目标发行版和所有者）中的箭头并选择所需值，为各相关字段设置值。注意：可以根据需要定制这些字段。
  4. 完成编辑之后，单击应用以保存更改。

现在就可以根据设置的值查询改进请求。

要查看有关对改进请求进行组织的更多信息，请参阅白皮书 Using Rational ClearQuest and Rational RequisitePro for Analysts（位于 Let's Go AnalystStudio 中）。

有关查询和定制字段的信息，请参阅 ClearQuest 联机帮助。

3. 运行查询

ClearQuest 提供了一个强大的查询界面，分析人员可使用该界面查询关联项目中的所有缺陷和改进请求。下面是可能有助于分析人员管理改进请求的查询。 分析人员可以运行以下查询：

  1. 将针对正在解决的特定发行版的改进请求隔离开来。由于并非收到的所有改进请求都可以在当前发行版中实施，因此使用“目标发行版”字段查询所有改进请求非常有用。该查询将返回查询中指示的发行版所触发和指定的所有改进请求的工作清单。
  2. 列出尚未与需求关联的所有改进请求。该查询将帮助分析人员查看已经在考虑的改进请求，或是已在排队但尚未分配给特定软件迭代的改进请求。
  3. 查找同一个改进请求的发生次数或实例数。该查询可帮助分析人员确定项目干系人将某个改进请求提交了多少次。这将对衡量权重流程有所帮助，以便确定该请求对项目干系人的重要性以及优先级级别。
  4. 区分真正的改进请求和对新功能的请求。该查询显示软件开发的进度。分析人员可就此分辨大多数改进请求是将修正现有功能，还是要求向当前产品添加新功能。

通过使用 ClearQuest 和 RequisitePro 集成，分析人员可以轻松地将已核准的当前版本改进请求与需求数据库中的有效需求相关联。

有关在 ClearQuest 中创建查询的更多信息，请参阅 ClearQuest 联机帮助。
