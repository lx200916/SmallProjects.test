工具向导：使用 Rational XDE Developer 合并现有设计元素
此工具向导描述如何执行“使用 Rational XDE Developer 合并现有设计元素”。
工具：Rational XDE Developer
关系
相关元素
  • 合并现有设计元素
主要描述

概述 

此工具向导支持使用 Rational XDE Developer - Java Platform Edition 合并现有设计元素。

  • 此工具向导中的步骤与相应任务中的那些步骤相匹配。
  • XDE 联机帮助中的主题链接标有 。
  • 在工具向导中，将执行以下步骤：
    • 确定复用可能性
    • 反向设计组件和数据库
    • 更新设计模型的组织
    • 更新逻辑视图

确定复用可能性

查找提供相似接口的现有子系统或组件。可能已使用可重用资产规范（RAS）封装了一些资产以供内部复用。如果使用 Rational XDE 创建，则此类资产可以在 XDE 中浏览和装入。请参阅 Getting Started with RAS Assets .

反向设计组件和数据库

对于不包含 XDE 模型的资产，反向设计模型以便更好地理解设计可能会很有帮助。如果该资产被证实为可使用，则 XDE 模型将成为您的总体设计模型的组成部分。 请参阅 Reverse Engineering .

对于您公司控制的资产，对候选接口做出较小的更改是有可能的，这将提高该接口与期望接口的一致性。对于您的公司未能控制的资产，您可以创建适配器或网桥设计元素，将您期望的接口映射到复用的资产提供的接口上。 请参阅工具向导：确定设计元素，获取关于创建和修改接口和类的指南。

更新设计模型的组织

按需要将设计元素重新组织到包中。请参阅 Grouping Model Elements in Packages . 另请参阅白皮书 Rational XDE Model Structure Guidelines。

更新逻辑视图

如果存在独立的逻辑视图，则需要对它进行维护。请参阅白皮书 Rational XDE Model Structure Guidelines。
