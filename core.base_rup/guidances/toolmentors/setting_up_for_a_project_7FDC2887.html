工具向导：使用 Rational XDE Developer 为项目进行设置
此工具向导描述如何为项目设置 Rational XDE 建模环境。
工具：Rational XDE Developer
关系
相关元素
  • 支持开发
主要描述

概述

为项目设置 XDE 是基础，这使团队成员能够同时对同一模型进行操作。开发复杂的系统要求成组的分析人员、设计人员和开发人员在进行各自工作的同时能够“纵观全局”。 成功管理多个团队成员对同一模型具有不同类型访问权的环境，则要求：

  • 阐明管理团队活动的工作策略
  • 具有能够支持该策略的工具

以下基本步骤奠定了这个基础：

  • 定制 XDE 模板和缺省值（可选）
  • 创建 XDE 项目和模型
  • 将 XDE 模型划分成受控单元

定制 XDE 模板和缺省值（可选）

在接下来的步骤中，您将使用标准模板创建 XDE 模型，这些模板包括符合 Rational XDE 模型构造指南 的基本封装结构。

您可以选择创建自己的专门模板。Rational XDE 允许您设置属性和选项，如缺省字体、颜色、行样式、文件存储选项、构造型和附加信息的显示方式以及应用的概要文件。

参考： Setting Up and Managing the Modeling Environment。

创建 XDE 项目和 XDE 模型

术语“项目”主要用于 RUP 中，它指的是整个软件开发工作。

在 Microsoft.NET 中，整个软件开发称为“解决方案”。“解决方案”由一个或多个 XDE 项目（更精确地说是 .NET 项目）组成，这些 XDE 项目中包含 XDE 模型、源代码以及其他文件。

在 Java Platform Edition 中，整个软件开发工作称为“应用程序”。J2EE 应用程序由一个或多个 XDE 项目组成，这些 XDE 项目中包含 XDE 模型、源代码以及其他文件。

类似地，RUP 中的“模型”工作产品是概念上的工作产品，可以由跨越多个 XDE 项目的多个 XDE 模型组成。

遵循 Creating Projects 中指定的步骤来创建 Rational XDE 模型构造指南 中指定的 XDE 项目。

遵循 Creating Models中的步骤 使用 Rational XDE 模型构造指南作为指导信息，根据需要向项目添加其他模型。

参考： Setting Up and Managing the Modeling Environment。

将 XDE 模型分成多个控制单元

XDE 支持将一个 XDE 模型划分成多个可管理的片段，这是通过允许您将一个 XDE 模型划分成多个称为受控单元的文件实现的。

请参阅： Working with Models Under Source Control。
