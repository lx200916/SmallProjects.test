工具向导：使用 Rational XDE Developer 正向设计数据库
此工具向导描述在 Rational XDE Developer 软件工具中如何使用正向设计，通过现有 Rational XDE 数据模型生成数据库模式或 DDL 脚本文件。
工具：Rational XDE Developer
关系
相关元素
  • 数据库设计
主要描述

概述

在本工具向导中执行以下步骤：

  • 设置模型属性
  • 将数据模型正向设计为数据库
  • 管理对模型和数据库的更改

设置模型属性

要启用数据模型的正向设计来创建数据库或数据定义语言（DDL）文件，必须设置两种类型的模型属性。这些属性为：

  • 缺省数据库目标数据库管理系统（DBMS）（请参阅Setting the Default Database Target ）
  • 缺省数据库指定（请参阅 Setting the Default Database Assignment ）

Rational XDE 支持对 ANSI SQL 92 标准和 Rational XDE 联机帮助主题 Databases Supported in Rational XDE 中列出的数据库管理系统（DBMS）进行正向设计 。

参考：设置数据建模选项 。

将数据模型正向设计为数据库 

Rational XDE 可用于将数据模型正向设计为新的数据库模式或新的 DDL 文件。您可以正向设计单独的表或整个数据模型。参考： Database and DDL Generation 可以使用 Rational XDE 提供的数据建模正向设计向导启动正向设计流程。（请参阅 Generating Databases or DDLs 。） 

无论您决定仅生成 DDL 还是执行 DDL 以创建数据库，Rational XDE 都将自动在您在向导中所指定的目录中创建一个 DDL 文件。Rational XDE 在正向设计流程中包含的元素取决于您在正向设计向导中选择的选项。所有元素或 DDL 脚本都是根据指定的 DBMS 生成的。请参阅以下 Rational XDE 联机帮助主题，获取有关受支持的不同类型的 DBMS 的详细信息：

  • DB2 Database and DDL Generation
  • Oracle Database and DDL Generation
  • SQL Server Database and DDL Generation
  • Sybase Database and DDL Generation

如果正向设计一个 ANSI SQL 92 数据模型，Rational XDE 将生成一个与 DBMS 无关的 DDL。可使用您所选的数据库工具执行该 DDL。

要执行数据库的正向设计，Rational XDE 必须建立到相应 DBMS 的连接。请参阅 Connecting to Database Management Systems 以了解更多关于建立数据库连接的信息。 请参阅以下 Rational XDE 联机帮助主题，获取有关为特定 DBMS 类型建立连接的信息：

  • Oracle Database Connections
  • SQL Server Database Connections
  • Sybase Database Connections
  • DB2 Database Connections

管理对模型和数据库的更改

创建了数据库或 DDL 之后，必须控制和管理对数据模型与生成的数据库的更改。Rational XDE 提供了一种比较和同步功能，用以帮助数据库设计员管理变更。 参考： Comparing and Synchronizing in Data Modeling 、工具向导：在 Rational XDE 中管理数据库。
