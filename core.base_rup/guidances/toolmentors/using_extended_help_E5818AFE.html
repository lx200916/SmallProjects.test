工具向导：通过 RUP Web 站点使用扩展帮助
此工具向导描述如何通过 Rational Unified Process Web 站点使用扩展帮助。
工具：Rational Unified Process
主要描述

概述 

Rational Suites 中的每个 Rational 工具都有扩展帮助。该工具“帮助”菜单中的该选项基于操作员所用的工具以及在该工具中的位置为操作员提供流程指导信息。

在本工具向导中执行以下步骤：

  • 调用扩展帮助
  • 复查过程指导信息
  • 可选：保存扩展帮助以供复查
  • 可选：选择其他 RUP 配置
  • 可选：在扩展帮助中添加更多主题
  • 更多信息

调用扩展帮助

从 Rational 工具的“帮助”菜单，选择“扩展帮助”。如果未出现扩展帮助，或者它呈灰色，则没有为您的计算机启用“扩展帮助”。这可能是由于您未购买 Rational 工具且未将其作为 Rational Suite 的一部分安装。

调用“扩展帮助”时，将出现一个对话框。该对话框将包含您已查找的所有 RUP 配置的列表，并询问您要将哪一个 RUP 配置用作过程指导信息。您可以选择缺省选项。

复查过程指导信息

RUP 浏览器将出现，在其前台有一个新的过程视图选项卡。其名称是“扩展帮助 － <工具名称>”，并且包含：与该工具中与上下文关系最紧密的 15 个主题；其余主题（按可能性降序排列）的“更多内容”文件夹；重新启动该对话框为扩展帮助选择 RUP 配置的选项。

您需要的信息很可能是“扩展帮助”选项卡中的前 15 主题中的一个。如果该工具具有工具向导，则它们可能会出现在前 15 个主题中。如果这些主题没有您感兴趣的内容，您可以打开“更多内容”文件，查看以相关性降序排列的其余主题。

“扩展帮助”将特定于您所选择的 RUP 配置。这样您就能够只获取关于您正在开发的项目的指导信息。

在任何时候，您都可以切换到其他某个流程视图、某个“个人流程视图”或“我的 RUP”选项卡，以获取进一步的信息。

可选：保存扩展帮助以供复查

当 RUP 浏览器关闭时，将删除会话期间创建的所有“扩展帮助”选项卡。如果您要保存某个“扩展帮助”选项卡以供以后复查，您可以使用“个人流程视图”或“我的 RUP”功能部件，以不同的名称保存它。单击树控件工具栏上的“另存为”按钮，并给出您希望的任何名称。

有用的提示是，当您复查主题时，您可以将它们拖动到“已复查”文件夹中，或者直接删除它们。这为您提供了跟踪复查内容的方法。类似地，通过将元素来回拖动或上下移动，您可以重新组织已保存的“个人流程视图”或“我的 RUP”选项卡中的元素。

可选：选择其他 RUP 配置

如果您意识到您正在查找错误的 RUP 配置环境中的“扩展帮助”，例如它适用于大型业务建模项目，但您当前正在开发一个小型 J2EE 项目，那么您可以选择其他 RUP 配置以从中获取流程指导信息。

在“扩展帮助”树控件的底部，有一个“选择其他扩展帮助”元素。选择它会重新启 RUP 选择对话框。 

可选：在扩展帮助中添加更多主题

“扩展帮助”完全基于在 Rational Method Composer 中所作的流程选择。将对任何选定的插件或进程组件建立索引，以供底层的“扩展帮助”上下文引擎使用。要获取更多插件，您可以访问 Rational Developer Network（www.rational.net）上的 RUP Plug-in Exchange。  

有关更多信息 

有关创建向扩展帮助添加主题的 RUP 插件的附加信息，请参阅 Process Engineering Process（PEP）。PEP 是类似于 RUP 的流程，它提供流程工程域的指导。
