工具向导：使用 Rational ClearCase 设置实施模型
此工具向导解释了如何使用 Rational ClearCase 来设置实施模型目录。
工具：Rational ClearCase
关系
相关元素
  • 构造实施模型
  • 建立配置管理（CM）环境
主要描述

术语

RUP 术语和 ClearCase 所用的术语之间存在一些差别。以下特定于工具的术语的定义将有助于阐明这些差别。

  • 配置管理：在 ClearCase 的环境中，术语“配置管理”的使用是指版本控制和工作版本管理。RUP 使用配置管理（CM）的 IEEE 和 ISO 定义，配置管理还包括变更管理。
  • 配置与变更管理：ClearCase 和 Rational ClearQuest 均指配置与变更管理，它是配置管理的 RUP 定义的同义词。RUP 以相同的含义使用这些术语。
  • 项目：ClearCase 项目映射到 RUP 项目存储库。

概述

本工具向导描述如何在开发环境中创建框架，以组织和存储与软件产品的实施和交付相关的配置项工作产品。开发环境中子系统的物理说明定义了“产品目录结构”，它实际上就是软件实施模型。

在开始使用 ClearCase 之前，需要知道您将使用 ClearCase 统一变更管理（UCM）这一现成可用的用途模型，还是使用基本 ClearCase，后者将提供一组可用于构造其他用途模型的工具，这很重要。

此工具向导适用于 ClearCase UCM 和基本 ClearCase 任务，而 RUP 中提供的所有其他 ClearCase 工具向导使用 UCM 模型。

此工具向导在运行 Microsoft Windows 时适用。

工具步骤

要使用 Rational ClearCase 设置实施模型：

  1. 创建配置管理（CM）存储库（ClearCase VOB）。
  2. 将现有文件和目录导入到 VOB 中。
  3. （可选）创建管理 VOB 以包含全局类型对象的定义。

1. 创建配置管理（CM）存储库（ClearCase VOB）

已确定版本的对象基础（VOB）存储文件、目录和其他对象的版本。它们充当配置管理信息的存储库。常见的做法是创建 VOB 以代表由体系结构定义的子系统。

要启动 VOB 创建向导（在向导的每一步均提供了上下文相关帮助。），请执行以下操作：

  1. 在 Windows 任务栏上，单击开始 > 程序 > Rational Software > Rational ClearCase > 管理 > 创建 VOB。
  2. 会启动 VOB 创建向导。遵循提供的步骤以创建 ClearCase VOB。在开始之前，知道站点或项目使用的 VOB 命名约定，以及放置 VOB 存储的位置，是有帮助的。
  3. 在向导的剩余步骤中，执行以下操作：
    • 命名 VOB 并指示它应是 UCM 组件还是项目 VOB（PVOB）。
    • 指定 VOB 存储位置。
    • 选择选项。

  有关创建 VOB 的详细信息，请参阅 ClearCase VOB 创建向导的联机帮助。

  有关设置 VOB 的概述，请参阅 ClearCase 手册 Administrator's Guide 中的“Setting Up ClearCase VOBs”一章。

2. 将现有文件和目录导入到 VOB 中

通过在为项目生成的 ClearCase 视图中操作，在 VOB 中创建所需的顶级目录结构，并将现有文件和目录复制到每个 VOB 中，以创建初始的开发配置项（Rational ClearCase 元素）集。

  1. 从 ClearCase Explorer 中，选择想要导入的目录和文件。 
  2. 右键单击以显示命令菜单，然后选择添加至源控制。这会为每个突出显示的对象创建 ClearCase 元素。

  有关导入大量文件的信息，请参阅 ClearCase 手册 Command Reference 中的 clearimport 命令。

3. （可选）创建管理 VOB 以包含全局类型对象的定义

（可选）您可能想要创建管理 VOB 来包含全局类型对象的定义。创建管理 VOB 以包含在子系统中定义基线时所用的全局类型对象的定义。

要创建管理 VOB，请启动先前在步骤 1 中描述的 VOB 创建向导，然后遵循提示。

  有关使用管理 VOB 的更多信息，请参阅 ClearCase 手册 Administrator's Guide 中的“Using Administrative VOBs and Global Types”。
