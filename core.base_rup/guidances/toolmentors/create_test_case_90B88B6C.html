工具向导：使用 Rational TestManager 创建测试用例
此工具向导描述如何使用 Rational TestManager 创建测试用例。
工具：Rational TestManager
关系
相关元素
  • 确定测试构想
主要描述

概述

 测试用例用于回答以下问题：“我需要测试的是什么？”测试用例提供测试工作的基础。测试用例共同组成了下列测试条件：

  • 测试内容
  • 测试方式
  • 测试时间

其中包含有关设计、需求验证、实施和配置的信息。

请将测试用例用作验证需求的基础，这可能基于各种来源的输入：Rational Rose 模型中的测试用例、规范、Rational RequisitePro 需求、营销间接材料、从代码复审收集的代码注释，以及变更请求。 

测试用例始终位于测试计划的测试用例文件夹中。有关如何创建测试计划的信息，请参阅工具向导：使用 Rational TestManager 创建测试计划。 

此工具向导适用于 Windows 98/ME/XP/2000 和 NT 4.0 平台。

工具步骤

要创建测试用例，请执行以下步骤：

  1. 插入测试用例
  2. 填写测试用例的属性

1. 插入测试用例

创建测试用例的第一部分为将一个测试用例插入 TestManager 中。 

  • 从“测试资产工作空间”的计划选项卡，单击文件 > 打开测试计划。
  • 选择相应的测试计划。 
  • 选择相应的测试用例文件夹，然后右键单击插入测试用例。将显示新建测试用例对话框。
  • 仅测试用例的名称为必需字段。请根据您的测试用例验证的内容对该测试用例命名，例如：“创建帐户”。在名称字段中，输入相应名称并单击确定。

  请参考 TestManager 联机帮助中的以下主题：

  • Inserting a test case folder into a test plan（目录：planning tests）
  • Inserting a test case into a test case folder（目录：planning tests）

2. 填写测试用例的属性

除指定测试用例名称之外，还可以指定其他属性。例如，可指定测试用例所有者，指定与该测试用例关联的配置和迭代，以及添加到与该测试用例关联的外部文档的指针。

测试用例属性可包含：
  • 测试用例的描述 － 该字段用于准确说明测试用例要验证的系统行为。
  • 测试用例的设计 － 这是说明测试用例如何执行指定测试（包括如何验证正确行为）的逐步指示信息。
  • 测试用例的所有者 － 通过测试用例来分配功能区域的所有权以构建团队。
  • 与测试用例关联的配置 － 指定需要执行测试用例以验证正确行为的硬件和软件配置。
  • 与测试用例关联的迭代 － 通过将测试用例与迭代关联，可使测试用例成为该迭代的验收条件的组成部分。从而可以帮助您了解何时需执行测试用例。
  • 与测试用例关联的测试输入 － 将测试输入与测试用例关联。测试输入是测试用例的激发因素。一个测试用例可能是验证测试输入的若干测试用例中的一个；例如，需求。测试输入来源非常广泛：Rational Rose 模型中的用例、规范、Rational RequisitePro 需求、营销间接材料、从代码复审收集的代码注释，以及变更请求。
  • 与测试用例关联的外部文档 － 外部文档可能包含任何内容，从详细的测试设计到用于实施测试用例的数据规范。
  • 测试用例的手动和／或自动实施 － 通过自动化测试脚本和／或手动脚本实施测试用例。
