工具向导：使用 Rational ClearCase 设置策略
本工具向导描述如何使用 Rational ClearCase 统一变更管理（UCM）来设置项目策略。
工具：Rational ClearCase
主要描述

概述

项目策略使得能在团队中强制实施好的开发实践。例如，设置策略能够将集成工作时可能遇到的问题减到最少。方法是：设置一个策略，要求开发人员在将工作交付到共享流之前用项目建议的基线来更新他们的工作空间。此做法减小了引入代码缺陷的可能性。

创建 UCM 项目时会设置项目策略。策略可在任何时候进行修改。

此工具向导在运行 Microsoft Windows 时适用。

工具步骤

为 UCM 项目设置策略有两种过程。

  • 为新项目设置策略
  • 为现有项目设置或更改策略

1. 为新项目设置策略

  • 从 Windows 任务栏，选择开始 > 程序 > Rational Software > Rational ClearCase > Project Explorer。
  • 从文件菜单，选择新建 > 项目以启动新建项目向导。得到提示时，请选择要为此项目强制实施的策略。

有关详细信息，请参考 Managing Software Projects 联机帮助。

2. 为现有项目设置或更改策略

  1. 从项目浏览器中，找到要修改的项目。
  2. 右键单击项目，并选择策略。
  3. 单击不同的选项卡以显示和选择项目策略。

有关每个项目策略的描述，请参考“策略”对话框的上下文相关帮助主题。

有关可以为 UCM 项目设置的策略的更多信息，请参考手册 IBM Rational ClearCase Guide to Managing Software Projects 中的“Setting policies”一章。
