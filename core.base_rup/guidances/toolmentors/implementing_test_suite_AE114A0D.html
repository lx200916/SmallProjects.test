工具向导：使用 Rational TestManager 实施自动化测试套件
此工具向导描述如何使用 Rational TestManager 设计功能测试或性能测试套件。
工具：Rational TestManager
关系
相关元素
  • 构造测试实施
  • 实施测试套件
主要描述

概述

套件显示任务的体系结构表示和要运行和测试的工作负载的体系结构表示。它显示计算机组、分配给各计算机组的资源、计算机组运行的测试脚本和每个测试脚本运行的次数等项。

在运行 Microsoft Windows 98/2000/NT 4.0 时此工具向导可用。

工具步骤

要使用 Rational TestManager 设计自动化测试套件，需执行以下步骤：

  1. 创建套件
  2. 向套件插入用户组（仅针对性能测试）
  3. 向套件插入计算机组
  4. 将测试脚本插入套件
  5. 向套件插入其他项

1.   创建套件

套件不仅可用于运行测试脚本，更重要的是，还可用于模拟使用系统的虚拟测试员的操作。套件可以简单到是执行一个测试脚本的一个虚拟测试员，也可以复杂到是不同组中的数百个虚拟测试员，而每个组还使用不同资源执行不同次数的不同测试脚本。

可以若干不同方式创建套件：

  • 使用性能测试套件向导
  • 使用功能测试套件向导
  • 基于任何类型的现有套件
  • 基于现有机器人会话
  • 使用空白的性能测试套件
  • 使用空白的功能测试套件

要使用这些方法的任何一种新建套件，请单击文件 > 新建套件。

注：使用向导新建套件时，必须具有可在套件中使用的有效测试脚本。

   请参阅 Rational TestManager 联机帮助中标题为 Creating a Suite 的主题。

2.   向套件插入用户组（仅针对性能测试）

用户组是所有性能测试套件的基本构建块。用户组是执行相同活动的虚拟测试员的集合。

  • 要向套件插入用户组，请从打开的套件中单击套件 > 插入 > 用户组。
  • 如下设置“用户”计数：

固定 － 指定固定数量的虚拟测试员。输入希望能够运行的最大虚拟测试员数。

可伸缩 － 指定动态数量的虚拟测试员。输入用户组代表的工作负载的百分比。

  • 如下设置计算机：

缺省计算机为 TestManager 本地计算机，但您可以指定用户组在定义的任何计算机上运行。

注：请将测试所需的任何用户创建的外部 C 库、Java 类文件或 COM 组件复制到“代理计算机”。

  • 也可以在多个计算机之间分发虚拟测试员。要在多个计算机之间分发一个用户组中的虚拟测试员，请单击套件 > 插入 > 用户组，然后单击多个计算机。

   请参阅 Rational TestManager 联机帮助中的 User groups into a suite 主题。

3.   向套件插入计算机组

计算机组是所有功能测试套件的基本构建块。实施测试的活动主要是创建可复用的测试脚本。 计算机组是运行相同测试脚本并且因而测试相同应用程序的一个或多个计算机或计算机列表。

  • 要向套件插入计算机组，请单击套件 > 插入 > 计算机组。
注：向套件添加计算机组时，您可以指定一个计算机组来用于运行时的资源提示，或指定大量计算机组来用于运行时的资源提示。但在一个套件内不能混用以下计算机组：带特定资源的计算机组与不带特定资源的计算机组。 
  • 要在一个计算机组中的多个计算机之间分发虚拟测试员，请单击套件 > 插入 > 计算机组，然后单击更改。

注：这样做的好处是：可通过在不同计算机上同时运行虚拟测试员来节约时间。

   请参阅 Rational TestManager 联机帮助中的 computer groups into a suite 主题。

4.   向套件插入测试脚本

要从打开的套件运行测试脚本，请为功能测试选择计算机组，或为性能测试选择用户组，然后单击套件 > 插入 > 测试脚本。

需要作出下列决策：

  • 测试脚本源 － 从中选择脚本类型（如 GUI、VU、VB、Java 等）。
  • 查询 － 从中选择要用于搜索脚本或采用新脚本的查询的类型。
  • 前置条件 － 这是对套件序列的提示，它表明对于您将运行的套件序列中的子项，测试脚本、套件或测试用例必须在该套件运行期间成功完成。
  • 迭代次数 － 所选脚本重复运行的频率如何？
  • 调度方法 － 显示项的情况以及这些项可用还是暂挂。

注意：不能在用户组中混用 GUI 测试脚本和 VU 测试脚本。不过，您可以混用其他测试脚本类型。

   请参阅 Rational TestManager 联机帮助中标题为 test scripts into a suite 的主题。

5.   向套件插入其他项

只要有计算机组和测试脚本，套件就可以运行。但是，比起该样本模型，对实际的虚拟测试员执行的工作进行实际建模的套件很可能更为复杂和不同。实际的套件可能还包含测试用例、子测试套件、场景、选择器（仅针对性能测试）、延迟、同步点和处理员（仅针对性能测试），以表示各种虚拟测试员操作。

可插入套件的其他项包括：

  • 插入测试用例
  • 插入套件
  • 插入场景
  • 插入选择器（仅针对性能测试）
  • 插入延迟
  • 插入处理员（仅针对性能测试）
  • 插入同步点

   请参阅 Rational TestManager 联机帮助中标题为 other items into a suite 的主题。

插入测试用例

  1. 要向套件插入测试用例，请从打开的套件中单击“套件 > 插入 > 测试用例”。
  2. 也可以对测试用例应用前置条件。请右键单击测试用例，然后选择运行属性。 

注：可将测试用例视为已配置，具体取决于其属性。

插入套件

要向套件插入套件，请从打开的套件中单击套件 > 插入 > 套件。

注：不能将基于用户组的性能套件放入另一个套件中。此外，插入套件中的基于计算机组的功能套件必须是使用为该计算机组选择的资源的提示选项创建的。

插入场景

场景可用于将测试脚本一起分到组中，从而可在多个用户组之间共享这些测试脚本。如果您有一个使用多个测试脚本的复杂套件，通过场景对这些测试脚本分组还能带来额外的好处：可以使您的套件更容易读取和维护。

  1. 要新建场景，请从套件的场景部分单击套件 > 插入 > 场景。
  2. 要向套件插入场景，请单击要放置场景的区域，然后单击套件 > 插入 > 场景。

注：在向用户组添加场景之前，最好填充场景。只要有测试脚本，场景就可以运行。但是，与用户组一样，实际的场景可能还包含选择器、延迟、同步点和处理员。场景甚至还可以包含其他场景。

插入选择器（仅针对性能测试）

与在套件中按照简单的顺序运行连续的项相比，选择器可以提供更为复杂的控制。选择器通知 TestManager 每个虚拟测试员按照何种顺序执行了哪些项。

要向套件插入选择器，请选择将包含此选择器的计算机组或场景，然后单击套件 > 插入 > 选择器。

选择器的类型包括：

  • 连续 － 按照套件中显示的顺序运行每个测试脚本或场景。
  • 并行 － 向可用的虚拟测试员分发测试脚本或场景（每台计算机一个虚拟测试员）
  • 随机可替换 － 选择器以随机顺序运行其下的项，每当选择了某项，再次选择此项的几率仍然相同。
  • 随机不替换 － 选择器以随机顺序运行其下的项，但每当选择了某项，几率就随之改变。
  • 动态负载均衡 － 如果使用此类型，将不随机选择项。将根据对这些项设置的权重进行选择，以平衡工作负载。可以针对时间或频率来平衡工作负载。

插入延迟

延迟通知 TestManager 要暂停多久才运行套件中的下一项。

要向套件插入延迟，请单击要添加延迟的计算机组（针对功能测试）、用户组（针对性能测试）、场景或选择器，然后单击套件 > 插入 > 延迟。

注：可将延迟插入套件或测试脚本。向套件插入延迟的好处是延迟在套件中是可见的，并且不编辑测试脚本就轻松更改延迟。

插入处理员（仅针对性能测试）

处理员通知 TestManager 给定时间段内每个虚拟测试员运行的任务数。

要向套件插入处理员，请选择将包含此处理员的用户组或选择器，然后单击套件 > 插入 > 处理员。

处理员可以为以下两种类型之一：

  • 协同处理员，它具有内置同步点，可用于指定您要达成的总速率。
  • 独立处理员，可使每个虚拟测试员（VT）独立运行。它不使用内置的同步点使其下的 VT 协同运行。

处理员可有以下两种速率之一：

  • 总速率 － 对于协同处理员，通常选择此速率。
  • 用户速率 － 对于独立处理员，必须选择此速率。

 对于处理员，分发选项有三种：

  • 常量分发表示每个事务完全按照您指定的速率发生。
  • 统一分发表示事务关于时间的平均速率等于您指定的速率（尽管每个事务之间的时间是常量）。
  • 相反，负指数分发则更改事务发生的概率。这种分发对于活动爆发之后逐渐减少这一典型的 VT 行为作了最为接近的模拟。

插入同步点

同步点使您可以通过在特定时间点（称为同步点）暂停各 VT 的执行来协调大量 VT 的活动。

要向套件插入同步点，请单击套件 > 插入 > 同步点。

同步点的设置包括：

  • 同时 － 同时释放所有虚拟测试员。
  • 交错式 － 逐个释放虚拟测试员。
  • 超时 － 表示同步点的该时间段指定的是 TestManager 等待 VT 到达同步点的总时间。超时周期在第一个 VT 到达同步点时开始。
