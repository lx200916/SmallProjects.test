工具向导：在 Rational Robot 中设置测试环境
此工具向导描述如何使用 Rational Robot 设置测试环境。
工具：Rational Robot
关系
相关元素
  • 执行测试套件
  • 支持开发
主要描述

概述

在运行 Microsoft Windows 98/2000/NT 4.0 时此工具向导可用。

要使用 Rational Robot 为测试设置测试环境，请执行以下操作：

  1. 为记录、编辑或测试脚本回放设置测试环境
  2. 设置 GUI 记录脚本选项
  3. 设置 GUI 脚本回放选项

1. 为记录、编辑或测试脚本回放设置测试环境

Microsoft Windows 环境和接受测试的应用程序的状态会影响脚本回放。如果在记录的环境和回放环境之间存在差异，那么回放时会发生问题。

在记录或编辑测试脚本前，请确保接受测试的应用程序和所有其他软件（在测试环境中）处于正确的初始状态。另外，在执行测试（回放测试脚本）之前，请确保接受测试的应用程序和所有其他软件处于与记录／编辑测试脚本时相同的初始状态。无论您开始记录时应用程序和窗口是否打开、活动或显示，当您开始回放时，该脚本应该为打开、活动或显示。另外，请确保任何相关网络设置、活动数据库和系统内存都处于与记录脚本时相同的状态。

回放测试脚本时，请确保设置正确的回放选项，包括 Rational Robot 应该如何处理意外的活动窗口和如何从脚本命令错误中恢复（请参阅下一部分）。

2. 设置 GUI 脚本记录选项

图形用户界面（GUI）脚本记录选项向 Rational Robot 提供在记录脚本期间如何拖动鼠标、处理某些对象、窗口设置、对象内容以及 Robot 窗口的指示信息。

要设置 GUI 脚本记录选项：

  1. 执行以下某个操作，显示“GUI 记录选项”对话框：
  • 在开始记录之前，单击工具 > GUI 记录选项。
  • 单击工具栏上的记录 GUI 脚本按钮开始记录。在“记录”对话框中，单击选项。
  1. 在每个选项卡上设置选项。有关每个选项卡的详细信息， 请参阅 Using Rational Robot 手册中的 Setting GUI Record Options 一章，或者 参阅 Robot 联机帮助。

常规 － 指定 Robot 如何识别列表和菜单内容以及不支持的鼠标拖动。 另外，从该选项卡上，您还可以指定：用于脚本自动命名的前缀、Robot 是否保存和恢复活动窗口的大小和位置、是否记录思考时间或者记录在按下 Enter 键后的延迟。

Robot 窗口 － 指定记录测试脚本期间如何显示 Robot 窗口。用户还能够在该选项卡上指定用于隐藏／重新显示 Robot 窗口以及在面向对象的记录和低级别记录方式之间切换的热键。

对象识别顺序 － 用于在记录前或记录时为个别对象类型更改对象识别方法的优先级。

有关如何记录脚本的信息，请参阅工具向导：使用 Rational Robot 创建测试脚本。

3. 设置 GUI 脚本回放选项

GUI 脚本回放选项为 Rational Robot 提供如何回放 GUI 脚本的指示信息。您可以在开始回放之前或回放过程的早期设置这些选项。

要设置 GUI 回放选项：

  1. 执行以下一个操作，显示“GUI 回放选项”对话框：
  • 在开始回放之前，单击工具 > GUI 回放选项。
  • 单击工具栏上的回放脚本按钮开始回放。在“回放”对话框中，单击选项。
  1. 在每个选项卡上设置选项。 

    有关每个选项卡的详细信息， 请参阅 Using Rational Robot 手册中的 Setting GUI Playback Options 一章，或者 参阅 Robot 联机帮助。

  2. 回放 － 指定命令和击键之间的延迟、是否使用记录的思考时间和输入延迟、是否跳过验证点、是否显示应答结果框以及回放期间 Robot 窗口中会发生什么事件。有关信息，请单击对话框中的帮助按钮。

    日志 － 指定日志中保存什么结果、日志是否在回放后自动出现、在覆盖日志前是否要提示您以及使用缺省日志信息还是您提供的信息。

    捕获匹配 － 指定 Robot 如何将记录期间捕获的窗口标题与回放期间找到的标题相匹配。有关信息，请单击对话框中的帮助按钮。

    等待状态 － 指定回放期间的缺省重试和超时值。

    意外的活动窗口 － 指定 Robot 在意外活动窗口出现时如何作出响应。

    错误恢复 － 指定 Robot 如何在脚本回放期间从脚本命令和验证点故障中恢复。

    陷阱 － 指定陷阱实用程序捕获关于回放过程中发生的常规保护故障的哪些信息以及如何恢复。

  3. 单击确定。

有关如何回放脚本的信息，请参阅工具向导：使用 Rational Robot 执行测试。
