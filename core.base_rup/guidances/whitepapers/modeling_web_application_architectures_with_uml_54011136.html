摘要

Web 应用程序变得日趋复杂和关键。为了帮助管理这种复杂性，需要对这些应用程序建模。UML 是针对软件密集型系统建模的标准语言。当尝试使用 UML 对 Web 应用程序建模时，很明显其中有些组成部分不能很好地符合标准 UML 建模元素。为了对整个系统（Web 组件和传统中间层组件）保持使用一种建模表示法，必须对 UML 进行扩展。本文提出了使用 UML 正式扩展机制的一种 UML 扩展。 设计该扩展是为了让特定于 Web 的组件可与系统模型的其余部分相集成，并显示适于 Web 应用程序设计人员、实施人员和设计人员的适当抽象和具体程度。
