任务：报告状态
此任务描述了如何提供关于项目状态的定期更新。
规程：项目管理
用途
  • 提供定期的项目状态更新，供“项目复审委员会”（PRA）复审
  • 将超出项目经理权限范围的问题上递给 PRA 以获得解决方案
关系
角色主执行者:
  • 项目经理
其他执行者:
输入必需:
  • 无
可选:
  • 风险列表
  • 状态评估
输出
  • 状态评估
流程使用情况
  • 项目管理  >  监视和控制项目  >  报告状态
步骤
准备状态评估
目的 记录当前项目状态，供 PRA 复审

状态评估可通过从 问题列表中提取的工作产品：状态评估和来自 任务：监视项目状态的 项目评估来传递。 “状态评估”应该论述以下方面：

技术进度： 该报告阶段中完成的工作（例如：完成的任务、交付的工作产品）。 突出强调任何落后状态。

对于当前的迭代，要报告为使迭代能按规划结束日期完成（即，让它限定时间）而必须作出的、在范围或质量方面的任何可能的变更（根据已发现的、将无法更正的缺陷进行报告）。

 
预算进度： 到目前为止的开支。突出强调任何超支的成本。 
预定里程碑的进度： 预定的里程碑实现了吗？ 
项目／产品的整个范围： 报告根据完成的工作而修改的项目范围评估以及对完成进展中的工作的评估。 
员工／人员安排状态： 员工的状态。报告任何问题或关心的事。 
风险状态： 有没有任何认识到的风险？ 
出现的问题： 需要获得 PRA 解决方案的项目问题。推荐可能的解决方案供考虑。 
操作任务： 一系列来自以前的状态评估的操作任务以及它们当前的状态。 

“状态评估”报告通常在“PRA 项目复审”期间报告给 PRA 并进行复审。
