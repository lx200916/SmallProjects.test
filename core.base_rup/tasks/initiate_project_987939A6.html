任务：启动项目
此任务描述了如何分配团队成员，该团队定义高级生命周期计划以及用来度量项目成功度的条件。
规程：项目管理

  • 项目经理

输入必需:
  • 业务案例

  • 无

主要描述

在“项目核准复审”核准了项目的业务案例之后，执行“启动项目”任务。此任务设置必要的执行管理和项目计划团队，还制定将用于确定项目何时已成功完成的条件。

步骤
指定项目复审委员会（PRA）

项目复审委员会（PRA）是负责检查项目的组织实体。 强烈建议任命个人作为 PRA，并指派一些人员（这些人员来自项目组织中已定义的一组资深技术人员和业务管理人员，以及执行官级别的客户方人员）协助他对项目进行监督。 对于中型合同软件项目，典型的协助 PRA 的小组可能包括以下人员：

  • 软件开发副总裁
  • 销售副总裁
  • 质量保证经理
  • 软件工程流程委员会代表
  • 定购软件的客户业务单位的经理
指定项目经理

项目委员会确定并核准一名具备适当技能和经验的项目经理。

指定项目计划团队

项目计划团队是初始的项目团队成员组，其中的成员将执行先启阶段的工作。 计划团队由项目经理与 PRA 一起确定、核准和指定。 项目计划团队通常可能包括以下人员：

  • 项目经理
  • 软件设计人员
  • 系统分析人员
  • 开发领导
  • 测试领导
  • 配置管理经理
  • 客户代表
核准产品验收条件

“启动项目”的最后一步是定义一些目标条件，这些条件将由客户用来确定何时可以接受可交付工作产品。这些条件应由客户组织和项目团队联合开发，可能包含以下各项：

  • 将所有确定为可交付工作产品的工作产品交付给客户
  • 列出验收测试的必需参与者
  • 必需的测试位置
  • 在产品验收计划中确定的工作产品评估的成功完成
  • 客户培训的成功完成
  • 现场安装的成功完成
  • 用于识别原始的项目规范已获满足的程度的度量
  • 用于识别业务案例的目标已获满足的程度的度量
